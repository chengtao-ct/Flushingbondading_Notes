### 🧠 DP 的第一层本质：对抗“指数爆炸”的降维武器

人类的直觉是“穷举（走一步看一步）”。但穷举在计算机里对应的是**树形结构**的不断分裂，稍微长一点的数组，就会产生 $2^N$ 种可能，这叫**指数爆炸**。

DP 的本质，是敏锐地发现：在这上亿种可能性里，**有大量的工作是在“重复造轮子”**。

举个最简单的例子：计算斐波那契数列 $F(5) = F(4) + F(3)$。

- **傻瓜穷举：** 为了算 $F(5)$，我去算 $F(4)$ 和 $F(3)$。但在算 $F(4)$ 的时候，我又去算了一遍 $F(3)$ 和 $F(2)$。你看，仅仅才第 5 层，$F(3)$ 这个状态就被**重复计算**了两次。如果到了第 50 层，底层的状态会被重复计算上百亿次！单片机当场死机。
    
- **DP 的灵魂操作（你说的记录）：** 我准备一个“记事本”。我第一次算完 $F(3)$，就把结果写在纸上。下次谁再需要 $F(3)$，**绝对不准再去算一遍，直接从纸上把数字抄走（提取）！**
    

这一个小小的“记录”动作，在数学图论上，把一颗庞大无比的**“树（Tree）”**，强行捏合、折叠成了一张简简单单的**“有向无环图（DAG）”**。把 $O(2^N)$ 的时间复杂度，直接降维劈成了 $O(N)$。

### 🧠 DP 的第二层本质：“无后效性”（极其冷酷的实用主义）

这是 DP 最隐秘、也最伟大的数学特质，学术界管它叫**马尔可夫性（Markov Property）**，大白话就四个字：**过去不问**。

昨天我们在讲“空两格”的时候，你是不是发现，你在决定“今天”要不要开机时，你**根本不关心**“前天”那个最大收益是怎么来的？你不在乎前天是为了拿到 10 而关了前面的 1，还是怎么着。

你眼里只有那个**最终的数字**。

这就叫“无后效性”：**“未来的决策，只取决于当前的状态，而与你是如何到达这个状态的路径无关。”**

正是因为有了这个冷酷的特性，你说的“比较 -> 记录”才敢放心大胆地执行。因为你确信，只要你记下了“截至到今天的最优解”，你就可以把之前所有的那些鸡毛蒜皮的纠结全部扔进垃圾桶。

### 🧱 1. `dp[i]` 的精准物理含义

在填表之前，你必须在心里给 `dp[i]` 下一个极其严酷的定义：

**`dp[i]` 绝对不是“第 $i$ 个模块的功耗”！**

**`dp[i]` 是：“从第 0 个模块一路走到第 $i$ 个模块，无论你中间怎么开怎么关，你能压榨出的【历史最高总功耗】”。**

既然是最高总功耗，那么走到第 $i$ 个模块时，你作为一个理性的机器，只有两条路可以走（而且必须二选一）：

### 🌌 2. 宇宙 A：为了保全大局，我选择【关闭】当前模块 $i$

- **现实情况：** 也许模块 $i$ 的功耗很小，但如果你强行开启它，就会导致前面那个超大功率的模块 $i-1$ 被迫关闭。
    
- **你的决策：** 忍痛割爱，当前模块 $i$ 坚决不开！
    
- **你的收益怎么算？** 既然第 $i$ 个模块你没碰，那你现在的总资产，就等于你走到上一个模块（$i-1$）时的总资产。
    
- **公式化：** **收益 = $dp[i-1]$**
    

### 🌌 3. 宇宙 B：收益实在太诱人，我选择【开启】当前模块 $i$

- **现实情况：** 模块 $i$ 的功耗巨大，开启它绝对血赚！
    
- **硬件物理限制（题意）：** 一旦你开启了模块 $i$，硬件保护机制会瞬间把相邻的模块 $i-1$ 给锁死（绝对不能开）。
    
- **你的收益怎么算？** 既然 $i-1$ 彻底废了，那你安全拿到手的历史最高资产，只能是倒退到 $i-2$ 那个位置时的总资产，再加上当前模块 $i$ 的功耗。
    
- **公式化：** **收益 = $dp[i-2] + nums[i]$

```c
int rob(int* nums, int numsSize) {
    // 1. 防呆拦截：防止 nums[1] 越界死机
    if (numsSize == 0) return 0;
    if (numsSize == 1) return nums[0];

    // 2. 初始化边界
    int prev2 = nums[0];
    int prev1 = nums[0] > nums[1] ? nums[0] : nums[1]; // C语言没有内置的max，用三目运算符最快

    // 3. 状态机流水线
    for(int i = 2; i < numsSize; i++) {
        // 计算今天
        int current = prev1 > (prev2 + nums[i]) ? prev1 : (prev2 + nums[i]);

        // 状态滚动 (注意顺序：先移 prev2，再盖 prev1)
        prev2 = prev1;
        prev1 = current;
    }
    
    // 4. 返回最终战果
    return prev1;
}
```
