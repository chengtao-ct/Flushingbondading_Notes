- **考察方向：** 多个链表指针的交替穿梭、**“哨兵节点（Dummy Node）”**的极其优雅的用法。
    
- **嵌入式场景：** 在自研 RTOS（实时操作系统）中，你有两个按时间戳排好序的“定时器任务队列”（单链表）。现在需要把它们合并成一个大的总队列交由 CPU 统一调度，且绝对不能申请新的节点内存，只能改变原有指针的指向。
    
- **挑战点：** 如何处理其中一个链表先被遍历完的情况？如何用“哨兵节点”完美避开极其繁琐的空指针 `if-else` 特判？



**思考**：
- 特殊情况先考虑：先处理两个都为空，再处理一个为空，最后是思考长短问题
- 卧槽这样还是很复杂啊，我去看了一下，Dummy-Node的介绍，我好像有点思路了



**问题**：
- 的确假如要处理空指针判断是很复杂的，如何简化，哨兵节点（Dummy Node）如何运用
- `&&` (逻辑与 / AND) —— “必须全是真”，`|` (逻辑或 / OR) —— “只要有一个真就行”
- 地基建立错误，就是本质上你没有创建一个新的列表，而是进行列表的拼接工作，还是对指针不理解的深入
- 其实是对链表的理解不够深入



```c
struct ListNode* mergeTwoLists(struct ListNode* list1, struct ListNode* list2) {
    // 1. 在栈上建一个实实在在的哨兵节点（打地基）
    struct ListNode dummy;
    dummy.next = NULL;
    
    // 2. 派 tail 游标站在地基上，准备开始拉拉链
    struct ListNode *tail = &dummy;
    
    // 3. 只要两个链表都没走到头，就不断比较
    while (list1 != NULL && list2 != NULL) {
        if (list1->val <= list2->val) {
            tail->next = list1;   // 缝合！把 tail 的尾巴接在 list1 上
            list1 = list1->next;  // list1 队伍往前走
        } else {
            tail->next = list2;   // 缝合！把 tail 的尾巴接在 list2 上
            list2 = list2->next;  // list2 队伍往前走
        }
        // 无论接了谁，tail 自己都要往前挪一步，永远站在新链表的最后面
        tail = tail->next; 
    }
    
    // 4. 循环结束，肯定有一个队伍空了。直接把没空的那个队伍整串挂在 tail 后面
    if (list1 != NULL) {
        tail->next = list1;
    } else {
        tail->next = list2;
    }
    
    // 5. 地基 dummy 的下一个节点，就是真正的链表头！
    return dummy.next;
}
```
