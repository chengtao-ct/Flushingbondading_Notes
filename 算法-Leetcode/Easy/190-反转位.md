- **考察方向：** 逐位提取、左移/右移、按位或（`|`）。
    
- **嵌入式场景：** 硬件通信协议中的**大小端转换**（Endianness）。比如，传感器通过 SPI 发来的数据是低位在前（LSB First），但你的单片机是高位在前（MSB First），你需要把这 32 个 bit 彻底倒个个儿。
    
- **挑战点：** 如何在不使用任何字符串或数组的前提下，纯靠 `>>` 和 `<<` 在一个 `uint32_t` 变量内部完成翻转。

### 正确的工业级心法：“左移腾地，按位或塞入”

在底层的位翻转操作中，我们通常不使用 `for(i=31 down to 0)` 这种复杂的下标计算，而是采用一种更优雅的**“流水线作业”**思维：

想象 `n` 是一个只能从**右边**出水的管子，`result` 是一个只能从**右边**进水的管子。 我们循环 32 次，每次只做三件事：

1. **腾空间：** 把 `result` 整体向**左**移 1 位 (`result <<= 1`)，最低位就空出了一个 `0`。
    
2. **滴水入海：** 提取 `n` 的最低位 (`n & 1`)，通过按位或（`|`）塞进 `result` 刚腾出的最低位里。
    
3. **推陈出新：** 把 `n` 向**右**移 1 位 (`n >>= 1`)，把刚才用过的最低位扔掉，让下一位顶上来。


```c
int reverseBits(int n) {

    uint32_t  result = 0;

  

    for(int i = 0; i <=31; i++)

    {

        result <<=1;

        result |=(n & 1);

        n >>=1;

    }

}
```

int reverseBits(int n) 
{
   int result = 0;
   int bit = 0;
   int j = 31;
   for(int i=0; i<32; i++)
   {
        bit = n&1;//提取最低位
        n = n >> 1;//前进
        result = result | (bit << j);//置位位第一位
        j--;
   } 
   return result;
}
