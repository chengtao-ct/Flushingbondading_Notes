# 91-工程源码索引

## 使用方法

这个文件只做工程入口索引，不展开函数细节。正文专题负责理解，索引负责回到源码。

## 内核公共入口

| 文件 | 用途 |
| --- | --- |
| `include/rtthread.h` | 用户可见内核 API、常用类型和对象声明 |
| `include/rtdef.h` | 基础类型、对象结构、链表、宏、配置相关定义 |
| `include/rthw.h` | 硬件抽象接口，如中断、上下文切换、CPU 相关接口 |

## 启动与自动初始化

| 位置 | 关注点 |
| --- | --- |
| `src/components.c` | `rtthread_startup`、`main_thread_entry`、自动初始化遍历 |
| `bsp/<board>/board.c` | `rt_hw_board_init`、heap 初始化、控制台绑定、板级设备准备 |
| `bsp/<board>/link.lds` 或 scatter 文件 | `__rt_init_start` / `__rt_init_end` 等段边界 |
| `applications/main.c` | 用户应用入口 |

## 对象系统

| 位置 | 关注点 |
| --- | --- |
| `src/object.c` | 对象初始化、动态分配、脱离、删除、查找、对象信息表 |
| `include/rtdef.h` | `struct rt_object`、`enum rt_object_class_type`、`struct rt_object_information` |
| `src/kservice.c` | 字符串、断言、基础服务函数，常被对象系统使用 |

## 线程模型

| 位置 | 关注点 |
| --- | --- |
| `src/thread.c` | 线程创建、启动、挂起、恢复、退出、延时 |
| `src/idle.c` | idle 线程、僵尸线程清理、空闲钩子 |
| `src/scheduler_comm.c` | 线程调度上下文初始化、时间片、优先级修改 |
| `libcpu/<arch>/` | `rt_hw_stack_init`、上下文切换汇编或架构适配 |

## 调度器

| 位置 | 关注点 |
| --- | --- |
| `src/scheduler_comm.c` | 调度公共逻辑，线程状态、tick、优先级辅助 |
| `src/scheduler_up.c` | 单核调度实现 |
| `src/scheduler_mp.c` | SMP 多核调度实现 |
| `include/rtdef.h` | 调度相关宏和线程状态定义 |
| `libcpu/<arch>/context*` | 上下文切换底层实现 |

## 中断、Tick、定时器

| 位置 | 关注点 |
| --- | --- |
| `src/irq.c` | 中断嵌套、进入/退出中断 |
| `src/clock.c` | 系统 tick 增加、tick 查询 |
| `src/timer.c` | 定时器对象、定时器链表、软/硬定时器 |
| `bsp/<board>/` | SysTick 或硬件 timer 如何调用内核 tick |

## IPC

| 位置 | 关注点 |
| --- | --- |
| `src/ipc.c` | semaphore、mutex、event、mailbox、message queue |
| `include/rtthread.h` | IPC API 声明和对象类型 |
| `src/thread.c` | IPC 等待时如何挂起线程 |
| `src/scheduler_*.c` | IPC 唤醒后如何进入就绪队列 |
| `src/timer.c` | IPC timeout 如何依赖定时器 |

## 内存管理

| 位置 | 关注点 |
| --- | --- |
| `src/mem.c` | 通用 heap / small memory 分配 |
| `src/memheap.c` | memheap 管理，适合多段内存 |
| `src/mempool.c` | 固定块内存池 |
| `src/slab.c` | SLAB 分配器 |
| `src/object.c` | 动态对象分配与释放 |
| `bsp/<board>/board.c` | 系统 heap 起止地址如何传入内核 |

## 建议阅读顺序

1. `src/components.c`：先建立启动主线。
2. `include/rtdef.h` + `src/object.c`：看对象系统和基础类型。
3. `src/thread.c`：看线程生命周期。
4. `src/scheduler_comm.c` + `src/scheduler_up.c`：看线程如何被调度。
5. `src/clock.c` + `src/timer.c`：看时间如何驱动状态变化。
6. `src/ipc.c`：看阻塞和唤醒闭环。
7. `src/mem*.c`：看动态内存和固定块分配。

## 不在第一阶段展开的入口

| 位置 | 放到第二阶段的原因 |
| --- | --- |
| `components/dfs/` | 依赖对象、设备、IPC、内存，适合内核核心之后读 |
| `components/finsh/` | 依赖对象枚举和设备控制台 |
| `components/net/` | 依赖线程、IPC、内存和设备驱动 |
| `components/drivers/` | 需要先理解对象系统和启动初始化 |
