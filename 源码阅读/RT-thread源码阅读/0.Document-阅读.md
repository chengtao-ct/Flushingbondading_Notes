
# Basic.md

1. RT-Thread 内核库也同样提供了此类函数的实现，主要供核心内核使用。不过，为了避免命名冲突，这些函数的名称特意加上了 rt_ 前缀
2. 线程调度算法采用的是**基于优先级的全抢占式多线程调度算法**
3. RT-Thread 支持创建多个具有相同优先级的线程。相同优先级的线程采用**时间片轮转调度算法 (Time Slice Rotation Scheduling)** 进行调度，以确保每个线程运行相同的时间。
4. RT-Thread 定时器提供两种定时机制：

- **单次触发定时器 (One-Shot Timer)** - 启动后仅触发一次定时器事件，随后自动停止。
    
- **周期触发定时器 (Periodic Trigger Timer)** - 周期性地触发定时器事件，直到用户手动停止，否则它将一直运行。
    

根据超时函数执行的上下文环境，RT-Thread 定时器可以设置为 HARD_TIMER（硬件定时器）或 SOFT_TIMER（软件定时器）模

5. 线程同步是通过获取和释放信号量与互斥量来实现的。

**互斥量 (Mutex)** 采用“优先级继承 (priority inheritance)”机制，有效解决了实时系统中常见的“优先级翻转 (priority inversion)”问题。线程同步机制允许线程按照优先级进行等待，或者按照先进先出 (FIFO) 的方式获取信号量/互斥量。

**事件集 (Event sets)** 主要用于线程间的同步，能够实现“一对多”和“多对多”的同步。它支持“或触发（独立同步）”和“与触发（关联同步）”，非常适合线程需要等待多个事件发生的情况。
6. RT-Thread 支持邮箱、消息队列等通信机制。邮箱中的消息长度固定为 4 字节。而消息队列则可以接收不同长度的消息，并将这些消息缓存在其自身的内存空间中
7. 在不同系统资源的情况下，动态内存堆管理模块会分别为内存规模较小的系统提供内存管理算法，为内存规模较大的系统提供 SLAB 内存管理算法
8. 还有一种名为 memheap 的动态内存堆管理方式，适用于那些具有不连续地址结构的系统。使用 memheap，用户可以将多个内存堆“拼接”在一起，使其像一个统一的内存堆那样被使用
9. RT-Thread 将 I2C、SPI、USB、UART 等作为外设设备，并通过统一的机制对这些设备进行管理。它实现了按名称访问的设备管理子系统，用户可通过统一的 API 接口来操作各种硬件设备。在设备驱动程序层面，根据嵌入式系统的特性，可以为不同设备绑定相应的事件。当设备事件发生时，驱动程序会通知上层应用程序。
10. 理解大多数代码通常都是从学习启动过程开始的，笑死我了，太有道理
11. `$Sub$$main` 可以调用一些需要在 main 函数之前执行的函数（此处添加了 RT-Thread 系统初始化函数）。之后，再调用 `$Super$$main` 来执行 `main()` 函数，这样用户就无需手动管理 main() 函数之前的系统初始化过程
12. 这段启动代码大致可以分为四个部分：

13. Initialize hardware related to the system.  
    初始化与系统相关的硬件。
14. Initialize system kernel objects, such as timers, schedulers, and signals.  
    初始化系统内核对象，如定时器、调度程序和信号处理机制。
15. Create the main thread initialize various modules in the main thread one by one.  
    在主线程中依次初始化各个模块。
16. Initialize the timer thread, idle thread, and start the scheduler.  
    初始化计时器线程和空闲线程，并启动调度器
