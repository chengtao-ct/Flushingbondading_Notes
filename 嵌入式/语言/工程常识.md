

extern "C" 的作用：禁用 C++ 的 Name Mangling

原理:
- C 编译器: 函数符号 = 函数名（无修饰）
- C++ 编译器: 函数符号 = 函数名 + 参数类型编码（支持重载）

问题: C++ 调用 C 库时，符号名不匹配导致链接失败
解决: 用 extern "C" 告诉 C++ 编译器按 C 规则生成符号

头文件模板:
#ifdef __cplusplus
extern "C" {
#endif
/* 函数声明 */
#ifdef __cplusplus
}
#endif

前缀

|前缀|含义|示例|说明|
|---|---|---|---|
|`g_`|Global 全局变量|`g_systemTick`|跨文件可见，慎用|
|`s_`|Static 静态变量|`s_rxBuffer[64]`|仅本文件可见，模块内部状态|
|`m_`|Member 类成员|`m_uartHandle`|C++/面向对象风格|
|`_`|私有成员（后缀风格）|`handle_`|Google风格用后缀下划线|
|`t_`|Thread-local 线程局部|`t_taskStackPtr`|RTOS中每个线程独立副本|
后缀

|后缀|含义|示例|
|---|---|---|
|`_t`|Type 类型定义|`Status_t`, `Error_t`|
|`_cb`|Callback 回调函数|`uart_rx_cb()`|
|`_isr` / `_irq`|中断服务程序|`TIM2_isr()`|
|`_handler`|事件处理函数|`dma_error_handler()`|
|`_init`|初始化函数|`peripheral_init()`|
|`_deinit`|反初始化|`peripher`|

continue vs break vs return

| 关键字    | 作用               | 跳转目标           |
|-----------|--------------------|--------------------|
| continue  | 跳过本次迭代       | 下一次循环判断     |
| break     | 跳出整个循环       | 循环后的代码       |
| return    | 退出整个函数       | 函数调用者         |

注意：
- break 只跳出最内层循环
- return 前需清理已申请的资源
- continue 适合"过滤-处理"模式，减少嵌套
