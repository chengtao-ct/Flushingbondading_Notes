CRC32 验证原理详解
本质：把数据当作一个巨大的多项式，除以一个固定的"除数"，得到"余数"
通俗理解：
CRC 就像"数据指纹"
┌─────────────────────────────────────────────────────┐
│  原始数据：                                          │
│  "Hello World" = 0x48 0x65 0x6C 0x6C 0x6F ...      │
│                                                     │
│  计算过程：                                          │
│  数据 ──► CRC算法 ──► 指纹值 = 0x1C291CA3          │
│                                                     │
│  存储：                                              │
│  ┌─────────────┬──────────────┐                    │
│  │ 数据        │ CRC指纹      │                    │
│  │ Hello World │ 0x1C291CA3   │                    │
│  └─────────────┴──────────────┘                    │
└─────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────┐

多项式除法原理：
把数据看作二进制多项式：
数据: 110100111
      ↓
多项式: 1·x⁸ + 1·x⁷ + 0·x⁶ + 1·x⁵ + 0·x⁴ + 0·x³ + 1·x² + 1·x¹ + 1·x⁰
除以固定多项式（CRC-32多项式）：
  110100111
÷ 100000111 (多项式)
───────────
  余数 = CRC
实际计算（二进制异或除法）：
        1011...  (商，被丢弃)
      __________
100000111|1101001110000... (数据后补32个0)
         100000111
         ──────────
          101010000...
          100000111
          ──────────
           01010111...
           ...
         ──────────
          余数 = CRC
代码逐行对应：
crc = 0xFFFFFFFF;        // 初始值，全1
for (每个字节) {
    crc ^= data[i];      // 把字节混入crc
    
    for (8位) {
        if (crc & 1) {   // 看最低位
            // 相当于多项式除法中这一位要"除"
            crc = (crc >> 1) ^ 0xEDB88320;  // 右移+异或多项式
        } else {
            // 这一位不需要"除"
            crc = crc >> 1;  // 只右移
        }
    }
}
return ~crc;  // 取反，得到最终CRC
为什么能检测错误？
CRC的数学特性：
1. 任意1位翻转 → CRC必定改变
2. 任意多位翻转 → CRC极大概率改变（99.9999%）
3. 多项式特性 → 检测突发错误能力极强
错误检测能力：

├────────────────────────────┤
│ 单位错误       100%        │
│ 双位错误       100%        │
│ 奇数位错误     100%        │
│ 突发错误<32位  100%        │
│ 随机错误       99.99...%   │
└────────────────────────────┘
