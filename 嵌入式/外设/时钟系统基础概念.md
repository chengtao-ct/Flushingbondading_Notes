# 时钟系统原理

## 概述

时钟是 MCU 的心跳，驱动所有时序逻辑电路的运行。时钟系统的核心任务是：**选择精度合适的时钟源，通过 PLL 倍频到目标频率，再通过总线分频分配给不同速度的外设，同时用时钟门控关掉不用的外设以省电。**

> [!info] 面试开场句
> "时钟系统本质是频率的变换和分配——晶振提供精度，PLL 提供频率，总线分频适配外设速度，时钟门控省电。"

---

## 时钟的本质

时钟 = 周期性的电脉冲信号。每来一个脉冲，电路做一步动作。

```
时钟脉冲:  |_‾|_‾|_‾|_‾|_‾|_‾|_‾|_‾|
CPU:       取指 译码 执行 取指 译码 执行 ...
定时器:    +1  +1  +1  +1  +1  +1 ...
UART:      采样 采样 采样 ...
```

**没有时钟 = 没有节拍 = 电路冻结 = 什么都不执行。**

| 频率越高 | 频率越低 |
|---------|---------|
| 执行越快 | 执行越慢 |
| 功耗越高 | 功耗越低 |
| 发热越大 | 发热越小 |

> [!tip] 这是所有 MCU 的通用规律，不限于 STM32。

---

## 四个时钟源

所有 MCU 的时钟源都来自两类：**内部 RC 振荡器** 和 **外部晶振**，按速度分高/低两组，构成 2×2 矩阵：

```
            高速（MHz 级）         低速（KHz 级）
内部 RC     HSI（精度差）          LSI（精度差）
外部晶振    HSE（精度高）          LSE（精度高）
```

### HSI（High Speed Internal）

- 来源：芯片内部 RC 振荡器，无需外部元件
- 频率：因芯片而异（STM32F1=8MHz, STM32F4=16MHz, ESP32 约不确定）
- 精度：差（±1%~3%），随温度漂移
- 用途：上电默认时钟，无晶振时的备选

### HSE（High Speed External）

- 来源：外部晶振（你焊在板子上的）
- 频率：由晶振决定（常见 4~26MHz）
- 精度：高（±10ppm~30ppm），几乎不漂移
- 用途：**主时钟源**，PLL 输入首选

### LSI（Low Speed Internal）

- 来源：芯片内部 RC
- 频率：约 17~40KHz（因芯片而异）
- 精度：差
- 用途：独立看门狗（IWDG），RTC 备选时钟

### LSE（Low Speed External）

- 来源：外部 32.768KHz 晶振
- 频率：32.768KHz
- 精度：高
- 用途：RTC 实时时钟

> [!tip] 为什么 RTC 用 32.768KHz？
> 2^15 = 32768，计数器溢出刚好 1 秒。这是全行业通用标准，所有 MCU 的 RTC 都一样。

### 为什么用 HSE 不用 HSI？

| 场景 | HSI 的问题 |
|------|-----------|
| UART 通信 | 精度 ±1%，波特率可能偏差超 2~3% 阈值 → 乱码 |
| USB 通信 | 要求精确 48MHz，HSI 精度不够 → 枚举失败 |
| 温度变化 | RC 振荡器随温度漂移 → 频率不稳定 |

**晶振提供精度，PLL 提供频率，两者配合 = 又准又快。**

> [!important] 通用结论
> 内部 RC 上电就能用（方便），但精度差；外部晶振精度高（可靠），但需要额外元件。所有 MCU 都是这个思路。

## PLL（锁相环）

### 解决什么问题？

HSE 精度高但频率低（比如只有 8MHz），CPU 需要跑 168MHz。PLL 就是把低频拉到高频的"变压器"。

### PLL 通用公式

```
F_out = F_in × N / M

M = 输入分频器（把输入频率降到 PLL 工作范围）
N = 倍频系数（VCO 输出频率）
P = 输出分频器（降到目标系统时钟）
Q = 另一路输出（常用于 USB 48MHz）

完整公式：
F_sys = F_in / M × N / P
```

### STM32F407 具体值（举例）

```
HSE = 8MHz

8MHz ÷ M(8) = 1MHz         ← PLL 输入（需在 1~2MHz 范围内）
1MHz × N(336) = 336MHz     ← VCO 输出（需在 100~432MHz 范围内）
336MHz ÷ P(2) = 168MHz     ← 系统时钟 SYSCLK
336MHz ÷ Q(7) = 48MHz      ← USB 时钟

每个芯片的 M/N/P/Q 范围不同，但原理完全一样。
查 Reference Manual 的 PLL 配置表。
```

> [!warning] PLL 有物理约束
> VCO 输入频率、VCO 输出频率都有范围限制，不是随便填的。必须查芯片手册的 PLL 配置表，超出范围会失锁（PLL 无法锁定 = 没有时钟输出 = 系统挂死）。

## 总线分频

CPU 和外设不需要跑同一个频率。时钟通过分频器分配给不同总线：

```
SYSCLK (168MHz)
    │
    ├── AHB 分频器 (/1)
    │   └── HCLK (168MHz) → CPU、DMA、Flash、GPIO
    │       │
    │       ├── APB1 分频器 (/4)
    │       │   └── PCLK1 (42MHz) → UART2~5、I2C、定时器2~7
    │       │
    │       └── APB2 分频器 (/2)
    │           └── PCLK2 (84MHz) → SPI、ADC、USART1、定时器1/8
    │
    └── ... 其他输出
```

### 为什么要分？

**1. 功耗** — 外设时钟频率越高功耗越大（P = CV²f），I2C 最大 400KHz 给 168M 时钟纯属浪费

**2. 硬件约束** — 不是所有外设电路都能跑高频，APB1 上外设设计上限就是 42M（F4）

> [!tip] 通用结论
> 所有 MCU 都有总线分频：CPU 高速总线，外设按速度分几组低速总线。名字不同但原理一样：
> - STM32：AHB / APB1 / APB2
> - ESP32：APB / AHB
> - NXP：AHB / IP Bus

### 面试常考的坑：定时器时钟倍频

当 APB 分频系数 > 1 时，总线上的定时器时钟自动 ×2：

```
APB1 分频 = /4 → APB1 总线 = 42MHz
定时器时钟 = 42 × 2 = 84MHz ← 计算 PWM 频率用这个！
```

> [!warning] 计算 PWM 频率时要用定时器时钟（84M），不是总线时钟（42M）。面试常考。

## 时钟门控（Clock Gating）

### 原理

MCU 上电后，默认所有外设的时钟都是关闭的。必须手动开时钟，外设才能工作。

```
RCC 时钟控制器
    ├── GPIOA  [开] → 工作，耗电
    ├── GPIOB  [开] → 工作，耗电
    ├── USART1 [开] → 工作，耗电
    ├── I2C1   [关] → 冻结，不耗电
    ├── SPI1   [关] → 冻结，不耗电
    └── ADC1   [关] → 冻结，不耗电
```

### 为什么不开时钟就不能用？

外设寄存器靠时钟驱动。时钟关了 → 寄存器没有驱动信号 → **读写操作无效**，外设相当于不存在。

### 为什么不全部常开？

80+ 个外设全开 → 即使不用也在耗电 → 功耗巨大。时钟门控 = 用哪个开哪个，不用就关掉。

> [!warning] 新手最常见的错误
> "我配置了 GPIO 但是引脚没反应" → 90% 是忘了开时钟。

> [!tip] 通用结论
> 时钟门控是所有低功耗 MCU 的标配。STM32 叫 RCC 使能，ESP32 叫 periph_module_enable，名字不同但本质一样。

## 时钟系统全景

```
                    ┌──── HSI (内部 RC)
                    │
晶振/RC ────────────┤──── HSE (外部晶振)
                    │
                    │         ┌── PLL 倍频 ──→ SYSCLK (系统时钟)
                    ├──── MUX ├
                    │  (选择器)├── 直通 ──────→ SYSCLK
                    │         │
                    │         └── HSI ────────→ SYSCLK (默认)
                    │
                    ├──── LSI (内部 RC) ──→ IWDG / RTC
                    │
                    └──── LSE (外部晶振) ──→ RTC

SYSCLK
  │
  ├── AHB 分频器 ──→ HCLK (CPU/DMA/GPIO)
  │     │
  │     ├── APB1 分频器 ──→ 低速外设 (UART/I2C/定时器)
  │     │
  │     └── APB2 分频器 ──→ 高速外设 (SPI/ADC)
  │
  └── MCO ──→ 时钟输出引脚（示波器观测用）
```

## 关键概念速查

| 概念 | 说明 | 通用性 |
|------|------|--------|
| 时钟源 | 提供原始频率的振荡器（RC/晶振） | 所有 MCU |
| MUX 多路选择器 | 选择用哪个时钟源 | 所有 MCU |
| PLL 锁相环 | 倍频，把低频拉到高频 | 大多数 MCU |
| 分频器 Prescaler | 把高频降到低速外设需要的频率 | 所有 MCU |
| 总线时钟 | AHB/APB 分级驱动不同速度的外设 | 所有 MCU |
| 时钟门控 | 关掉不用的外设时钟省电 | 所有低功耗 MCU |
| MCO | 时钟输出引脚，示波器可观测 | STM32 特有 |

## 面试高频问题

> [!example]- Q1：MCU 为什么需要时钟？
> 时钟是时序逻辑电路的驱动信号，没有时钟所有电路冻结不工作。频率越高执行越快但功耗越大。

> [!example]- Q2：HSI 和 HSE 的区别？为什么用 HSE？
> HSI 是内部 RC 振荡器，精度差 ±1%，会随温度漂移；HSE 是外部晶振，精度高 ±10ppm。UART/USB 等需要精确时序的通信必须用 HSE，否则会乱码或枚举失败。

> [!example]- Q3：PLL 解决了什么问题？
> 晶振频率有限（通常几 MHz~几十 MHz），但 CPU 需要跑更高频率。PLL 通过倍频把低频拉到高频。公式 F_out = F_in × N / M，具体参数查芯片手册的 PLL 配置表。

> [!example]- Q4：为什么总线要分频？不分行不行？
> 外设速度不同，全跑 168M 浪费功耗且有些外设电路不支持那么高的频率。低速外设给低频时钟省电，高速外设给高频时钟满足性能。

> [!example]- Q5：APB1 总线 42MHz，定时器时钟为什么是 84MHz？
> 当 APB 分频系数 > 1 时，定时器有自动 ×2 倍频。所以 APB1 总线 42MHz，定时器时钟实际 84MHz。计算 PWM 频率必须用 84MHz。

> [!example]- Q6：什么是时钟门控？为什么要开时钟才能用外设？
> 上电默认所有外设时钟关闭以省电。外设寄存器靠时钟驱动，关了时钟读写无效。"配置了外设但不工作"最常见的原因就是忘了开时钟。
