二、4MB Flash — 存放什么？
Flash 是 ESP32 的 *"硬盘"*，断电不丢失，所有持久化数据都在这里：
4MB Flash 物理地址空间（0x000000 ~ 0x3FFFFF）
┌──────────┬─────────────┬──────────┬─────────────────────────────────┐
│ 偏移地址   │ 大小         │ 内容      │ 作用                            │
├──────────┼─────────────┼──────────┼─────────────────────────────────┤
│ 0x000000 │ 4KB         │ MBT      │ Master Boot Record（出厂已写）    │
│          │             │          │ 存储Flash启动模式、频率等参数     │
├──────────┼─────────────┼──────────┼─────────────────────────────────┤
│ 0x001000 │ 16~20KB     │Bootloader│ 二级引导程序（你自己编译烧录的）   │
│          │             │          │ 负责读取分区表、加载APP、验证签名 │
├──────────┼─────────────┼──────────┼─────────────────────────────────┤
│ 0x008000 │ 3KB         │Partition │ 分区表                           │
│          │             │ Table    │ 告诉Bootloader各区域在哪、多大   │
├──────────┼─────────────┼──────────┼─────────────────────────────────┤
│ 0x010000 │ 1.8~3MB     │APP固件   │ 你写的应用程序（最大的区域）     │
│ (0x110000)│             │          │ 包含代码段(.text)+只读数据(.rodata)│
│          │             │          │ +FreeRTOS + Wi-Fi协议栈 + 你的代码│
├──────────┼─────────────┼──────────┼─────────────────────────────────┤
│ 0x310000 │ 1MB         │SPIFFS/   │ 文件系统区域                     │
│ (示例)   │             │NVS/FatFS │ 存网页文件、配置文件、OTA数据     │
├──────────┼─────────────┼──────────┼─────────────────────────────────┤
│ 0x9000   │ 16KB        │NVS       │ Non-Volatile Storage             │
│          │             │          │ 键值对存储：Wi-Fi密码、校准数据   │
├──────────┼─────────────┼──────────┼─────────────────────────────────┤
│ 0x4000   │ 8KB         │PHY数据   │ Wi-Fi/蓝牙 校准参数              │
└──────────┴─────────────┴──────────┴─────────────────────────────────┘
各区域详细说明
① 0x000000 — MBT（Master Boot Table）
这块出厂时 Espressif 已经烧好了，你一般不用管。
内容：
├── Flash 访问模式（DIO/DOUT/QIO/QOUT）
├── Flash 频率（40MHz/80MHz）
├── 芯片型号标识
└── 引导日志（Boot log）
作用：芯片上电后，ROM Bootloader 先读这里，
      知道 Flash 怎么访问、从哪加载 Bootloader。
② 0x001000 — Bootloader（二级引导）
你的项目编译后生成的 bootloader.bin
作用：
1. 初始化 SPI Flash（切换到高速模式）
2. 读取 0x8000 处的分区表
3. 根据分区表找到 APP 分区
4. 验证 APP 的 SHA256/签名（如果启用了 Secure Boot）
5. 将 APP 的 .text 段映射到内存
6. 跳转到 APP 的入口地址执行
为什么需要它？
  ROM Bootloader 太简单，只能加载一个固定地址。
  有了二级 Bootloader 才能支持：
  - 多分区（OTA A/B 切换）
  - Flash 加密
  - 安全启动
  - 自定义分区布局
③ 0x008000 — 分区表
分区表本身就是一个 "目录索引"：
| # | Name     | Type  | SubType | Offset   | Size   |
|---|----------|-------|---------|----------|--------|
| 1 | nvs      | data  | nvs     | 0x9000   | 16KB   |
| 2 | phy_init | data  | phy     | 0xf000   | 4KB    |
| 3 | factory  | app   | factory | 0x10000  | 3MB    |
| 4 | storage  | data  | spiffs  | 0x310000 | 1MB    |
作用：Bootloader 读这张表，才知道：
  - APP 固件在哪个偏移地址
  - NVS 数据区在哪
  - 文件系统区在哪
  - 有没有 OTA 分区（A区/B区）
④ 0x010000 — APP 固件
你的应用程序，编译后的 .bin 文件，通常 1~3MB
内部结构：
├── .text      → 可执行代码（CPU指令）
│                包括：你的代码 + FreeRTOS + Wi-Fi协议栈 + 驱动
│
├── .rodata    → 只读常量数据
│                包括：字符串常量、查表、摄像头驱动参数
│
├── .data      → 已初始化的全局/静态变量（启动时从Flash复制到SRAM）
│
└── .bss       → 未初始化的全局/静态变量（启动时在SRAM中清零）
                 （.bss本身不占Flash空间，只记录大小）
⑤ 0x9000 — NVS（Non-Volatile Storage）
键值对存储，类似 Android 的 SharedPreferences
典型存储内容：
├── Wi-Fi SSID + 密码
├── 蓝牙配对信息
├── PHY 校准数据（每块芯片出厂不同！）
├── OTA 分区状态（标记当前从哪个分区启动）
├── 用户自定义配置（设备ID、服务器地址等）
└── Boot count（启动次数计数）
特点：
├── 通过 nvs_set_str / nvs_get_str 读写
├── 内部使用磨损均衡（Flash 有擦写寿命）
└── 即使 OTA 升级，NVS 数据不丢

三、520KB SRAM — 存放什么？
SRAM 是 ESP32 的 *"运行内存"*，速度极快但断电丢失：
520KB 内部 SRAM 布局
┌──────────────────────────────────────────────────┐
│                                                  │
│  ┌─────────────────────────────┐  约 320KB       │
│  │  Free Heap（动态内存堆）     │                  │
│  │  ├── malloc() / free()      │                  │
│  │  ├── 任务栈（每个任务的栈）  │                  │
│  │  ├── 摄像头小帧缓冲         │                  │
│  │  ├── Wi-Fi 接收缓冲         │                  │
│  │  └── 你的动态分配数据       │                  │
│  └─────────────────────────────┘                  │
│                                                  │
│  ┌─────────────────────────────┐  约 100KB       │
│  │  .data 段（已初始化全局变量）│                  │
│  │  例如：int count = 42;       │                  │
│  │  启动时从 Flash 复制过来     │                  │
│  └─────────────────────────────┘                  │
│                                                  │
│  ┌─────────────────────────────┐  约 100KB       │
│  │  .bss 段（未初始化全局变量） │                  │
│  │  例如：static int buffer[8192];│               │
│  │  启动时自动清零              │                  │
│  └─────────────────────────────┘                  │
│                                                  │
│  ⚠️ 实际可用堆 ≈ 200~250KB（其余被系统占用）     │
│                                                  │
└──────────────────────────────────────────────────┘
SRAM 里的典型消耗者
Wi-Fi 协议栈：    ~50KB
FreeRTOS：        ~10KB（内核 + IDLE 任务 + Timer 任务）
TCP/IP 栈 (lwIP)：~40KB
蓝牙（如果开启）： ~60KB
摄像头驱动：      ~20KB
你的代码栈+堆：   剩余部分
──────────────────────
总需求轻松超过 300KB！所以 ESP32-CAM 必须用 PSRAM 补充

四、448KB ROM — 存放什么？
ROM 是芯片出厂时 光刻写入 的，永远不可修改：
448KB 内部 ROM 布局
┌──────────────────────────────────────────────────┐
│                                                  │
│  ┌─────────────────────────────┐                 │
│  │  一级 Bootloader             │                 │
│  │  芯片上电第一条执行的代码     │                 │
│  │  └─ 读取 Flash MBT           │                 │
│  │  └─ 加载 0x1000 处 Bootloader│                 │
│  └─────────────────────────────┘                 │
│                                                  │
│  ┌─────────────────────────────┐                 │
│  │  基础加密算法库               │                 │
│  │  ├── AES 加解密              │                 │
│  │  ├── SHA-256 / SHA-512       │                 │
│  │  ├── RSA 验证                │                 │
│  │  ├── ECC 椭圆曲线            │                 │
│  │  └── 随机数生成器            │                 │
│  └─────────────────────────────┘                 │
│                                                  │
│  ┌─────────────────────────────┐                 │
│  │  基础外设驱动                 │                 │
│  │  ├── UART 最小驱动           │                 │
│  │  ├── SPI Flash 读取          │                 │
│  │  └── GPIO 基础操作           │                 │
│  └─────────────────────────────┘                 │
│                                                  │
│  ┌─────────────────────────────┐                 │
│  │  芯片校准数据（每颗芯片唯一） │                 │
│  │  ├── MAC 地址（Wi-Fi/BT）    │                 │
│  │  ├── ADC 校准曲线            │                 │
│  │  └── Flash 电压参数          │                 │
│  └─────────────────────────────┘                 │
│                                                  │
│  🔒 只读！你永远无法写入或修改 ROM                 │
│                                                  │
└──────────────────────────────────────────────────┘
ROM 的特殊性
为什么需要 ROM？
├── 没有ROM，芯片上电不知道该干什么（Flash里什么都还没烧）
├── ROM 保证任何状态下都能通过 UART 恢复（救砖）
└── 加密算法固化在ROM里 → Secure Boot 无法被篡改
ROM vs Flash vs SRAM 对比：
┌────────┬─────────┬──────────┬──────────┬───────────┐
│        │ ROM     │ Flash    │ SRAM     │ PSRAM     │
├────────┼─────────┼──────────┼──────────┼───────────┤
│ 位置   │ 芯片内部│ 芯片外部  │ 芯片内部 │ 芯片外部  │
│ 可写   │ ❌      │ ✅       │ ✅       │ ✅        │
│ 速度   │ 快      │ 慢(需映射)│ 最快     │ 较慢      │
│ 断电   │ 保留    │ 保留     │ 丢失     │ 丢失      │
│ 容量   │ 448KB   │ 4MB      │ 520KB    │ 4MB       │
│ 存什么 │ 固化代码 │ 固件+数据 │ 运行时   │ 大缓冲区  │
└────────┴─────────┴──────────┴──────────┴───────────┘

五、4MB PSRAM — 存放什么？
PSRAM（Pseudo-SRAM，伪静态RAM）
本质是一块通过 SPI 接口连接的外部 RAM
┌──────────────────────────────────────────────────┐
│  4MB PSRAM 布局                                   │
│                                                  │
│  摄像头帧缓冲（最大消耗者）                        │
│  ├── 双帧缓冲：fb1 = 3.75MB (UXGA)               │
│  │              fb2 = 3.75MB                      │
│  │   → 放不下！所以 UXGA 实际用 JPEG 模式压缩     │
│  │   → VGA 直出 = 640×480×2 = 600KB ✅ 可以       │
│  │                                                │
│  ├── JPEG 编码输出缓冲                            │
│  │   → UXGA JPEG 一张 ≈ 40~100KB                 │
│  │                                                │
│  └── 图像处理临时缓冲                             │
│                                                  │
│  大块动态分配（溢出到 PSRAM 的 malloc）           │
│  ├── 大数组、JSON 解析缓冲                        │
│  ├── HTTP 响应缓冲                                │
│  └── WebSocket 发送队列                           │
│                                                  │
│  ⚠️ 速度比内部 SRAM 慢 ~10倍                      │
│  ⚠️ DMA 不能直接访问 PSRAM（部分操作有限制）      │
│  ⚠️ 中断处理函数不要用 PSRAM 里的数据              │
└──────────────────────────────────────────────────┘

六、完整启动流程中各存储器的协作
上电瞬间
│
├─① CPU 从 ROM 地址 0x40000000 开始执行（ROM Bootloader）
│     ROM 内部 │
│
├─② ROM 读取 Flash 0x000000 处的 MBT
│     Flash 头部 │
│     → 得知 Flash 模式/频率/芯片型号
│
├─③ ROM 加载 Flash 0x001000 处的 Bootloader 到 SRAM
│     Flash → SRAM │
│     → 跳转到 SRAM 中的 Bootloader 执行
│
├─④ Bootloader 初始化 PSRAM（如果配置了）
│     PSRAM 映射 │
│
├─⑤ Bootloader 读取 Flash 0x008000 处的分区表
│     Flash 分区表 │
│     → 知道 APP 在 0x010000
│
├─⑥ Bootloader 将 APP 的 .text 映射到 CPU 地址空间
│     Flash APP → CPU 通过 Cache 直接执行（XIP方式）
│     .data 段从 Flash 复制到 SRAM
│     .bss 段在 SRAM 中清零
│
├─⑦ 启动 FreeRTOS
│     SRAM 中运行 │
│     → 创建 idle 任务、timer 任务
│
└─⑧ 调用 app_main()
       你的代码开始执行！

七、一句话总结
存储器	一句话
ROM	出厂固化，芯片上电第一条指令的来源，永远改不了
Flash	外部"硬盘"，存固件、分区表、NVS、文件系统，断电不丢
SRAM	内部"内存"，跑代码的运行时数据，断电即丢，非常紧张
PSRAM	外部"扩展内存"，专门补 SRAM 不够用的场景（摄像头帧缓冲）
