## 【核心三件套的深度剖析】

### 信息含量递减图

```mermaid
flowchart TB
    subgraph ELF[".elf (最完整)"]
        E1["机器码 (.text等)"]
        E2["段信息 (地址/大小)"]
        E3["符号表 (函数/变量)"]
        E4["调试信息 (行号/DWARF)"]
    end

    subgraph HEX[".hex (带地址)"]
        H1["数据字节 (文本格式)"]
        H2["目标地址 (每行自带)"]
        H3["校验和 (和校验)"]
    end

    subgraph BIN[".bin (最纯粹)"]
        B1["纯字节流"]
        B2["无地址、无符号、无校验"]
    end

    ELF -->|"objcopy -O ihex"| HEX
    HEX -->|"objcopy -O binary"| BIN

    style ELF fill:#e1f5fe
    style HEX fill:#fff3e0
    style BIN fill:#f3e5f5
```

### 三者的本质区别

| 特性 | .elf | .hex | .bin |
|------|------|------|------|
| **地址信息** | 段头表完整记录 | 每行自带地址 | 无，需外部指定 |
| **符号信息** | 完整符号表 | 无 | 无 |
| **调试信息** | DWARF完整 | 无 | 无 |
| **空洞处理** | 支持（.bss段） | 支持（跳过不写） | 填充0x00或0xFF |
| **典型大小** | 最大 | 中等（文本编码+开销） | 最小 |
| **主要用途** | 调试、分析 | 烧录、量产 | OTA、Bootloader |
