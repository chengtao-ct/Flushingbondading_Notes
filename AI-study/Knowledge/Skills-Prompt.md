# 智能体工程核心：Prompt 与 Skills 的本质区别与协同架构

在 AI Agent（如 OpenCode、Claude Code）的开发与使用中，从“聊天窗口”走向“自主运行”，核心的范式转变在于如何管理 AI 的行为边界。这不仅是写提示词的技巧，更是系统架构的设计。

本文档梳理了 **Prompt（系统提示词）** 与 **Skills（技能系统）** 的底层逻辑、核心差异以及它们如何在工程中协同工作。

---

## 1. 核心定义与本质洞察

### Prompt（系统提示词）：全局的“宪法”与基调
* **本质**：强行设定的全局执行上下文。它在系统运行时必须时刻被遵守。
* **核心作用**：设定 AI 的“性格”、默认语言、总体思维框架（如：“你是一个遵循 MISRA-C 标准的资深工程师”）。
* **局限性（中间遗忘效应）**：如果将所有复杂的工程规范（如架构要求、防错指南）全部塞入 Prompt，会导致 AI 的上下文窗口过载。在处理长任务时，AI 会出现“注意力涣散”，忽略掉其中大部分的防御性规则，从而产生幻觉。

### Skills（技能系统）：风险监管与动态应对预案
* **本质**：按需动态加载（RAG）的上下文外挂，是智能体的“断路器”。
* **核心作用**：在节省 Token 的同时，对可能产生高风险的区域（如修改底层驱动、操作 Docker 容器网络）进行强制限制与风险应对。
* **优势**：它不是时刻生效的，而是通过特定的触发条件（描述中的 `Use when...`）唤醒。保证了在任何一个高危时刻，AI 脑容量里装的都是纯度 100%、绝对相关的最高优先级 SOP（标准作业程序）。

---

## 2. 嵌入式架构类比（The Mental Model）

为了更直观地理解，我们可以将 AI Agent 映射为典型的底层硬件/嵌入式系统架构：

| 概念 | 嵌入式系统映射 | 运行机制与表现 |
| :--- | :--- | :--- |
| **Prompt** | **`while(1)` 主循环 / RTOS 全局配置** | **常驻后台**。维持系统的基础运转与默认状态。如果主循环过于臃肿，会导致系统处理任务时掉帧、跑飞甚至死机。 |
| **Skills** | **ISR（中断服务函数）/ 硬件看门狗** | **按需触发**。平时休眠，一旦 AI 的行为触碰“高风险区域”（如尝试重启容器、修改引脚映射），立刻触发最高优先级中断，强行切入严密的防错 SOP。处理完毕后退出中断，返回主线。 |

---

## 3. 详细对比矩阵

| 维度 | Prompt (系统提示词) | Skills (外挂技能系统) |
| :--- | :--- | :--- |
| **生效范围** | 全局 (Global) | 局部/特定场景 (Local/Contextual) |
| **Token 消耗** | 持续消耗，极高（随对话无限累积） | 仅触发时消耗，极低（即插即用） |
| **管理目标** | 设定专家级的人设与通用思考路径 | 建立高危操作的验证门（Gatekeeper） |
| **开发方式** | 散文式编写，一次性喂给大模型 | TDD 测试驱动（发现漏洞 -> 编写极简规则堵漏 -> 测试） |
| **适用场景** | 聊天教学、系统性梳理知识体系 | 自动操作本地代码库、复杂环境配置、高风险重构 |

---

## 4. 协同工作流：安全地进行 Vibe Coding

在实际的工程开发中，高手的做法是将两者结合，打造一个既能放飞自我，又绝对安全的“护城河”环境。

1. **基调奠定（Prompt）**：用 Prompt 告诉 AI “你现在是一个全栈架构师，我们要开发一个高并发的自动化脚本系统”。
2. **排雷固化（Skills）**：梳理出架构中的脆弱点（如 DOM 加载延迟、容器通信端口异常），并把对应的防御措施写成 `SKILL.md`（例如规定：*每次生成浏览器操作代码前，必须确认显式等待逻辑*）。
3. **无脑狂飙（Vibe Coding）**：在有了 Skills 作为底层“看门狗”之后，开发者可以用最随意的自然语言下达指令。一旦 AI 试图写出危险代码，Skills 会自动跳出来拦截并强制它自我审查。

**结论**：Prompt 让你享受和顶尖大脑对话的快感；而 Skills 确保这个顶尖大脑在落地干活时，不会像个莽撞的实习生一样搞砸你的项目。
