# 生成式人工智能

- **基于概率统计的自回归预测系统**
- 就是下一下token的概率预测和选择
- 也就是所谓的文字接龙，按长文本不停的预测下一个字

# Token

- 是系统处理信息的最小原子单位
- 万物皆可 Token：跨模态的统一
- **Vocabulary**：Token 的集合,Llama 3.2 为例，其词表规模约为 128,000 个 Token

# Top-k Sampling
为了兼顾“正确性”与“多样性”，目前的工程主流是 **Top-K Sampling**。即：仅允许机率最高的前 K 个 Token 参与“掷骰子”，彻底屏蔽掉那些机率特别低的噪声 Token

# Tokenizer
- 它是将连续的现实世界信息映射到离散数学空间的**特征预处理器**
- 核心功能：**Encoding 与 Decoding**
- Encoding（编码）:编码器通常会自动添加**特殊 Token**（如代表句首的 `BOS` 或 ID `128000`），以标记序列的物理边界


# Prompt

## 工程干预：Prompt 的三层结构

在实际生产中，你看到的 Prompt 往往是经过封装的“复合体”，主要包含以下三层：

• **User Prompt（用户提示词）**： 这是最表层的输入，即用户直接输入的文字。

• **System Prompt（系统提示词）**： 这是开发人员预设的“隐形约束”。例如，为了让模型知道自己的身份或当前日期，系统会在用户输入前强行注入：“你的名字是 Llama，今天是 2025 年 9 月 12 日”。模型正是基于这些隐藏在 Prompt 里的信息，才能准确回答日期问题。

• **Chat Template（对话模板）**： 为了解决“模型只会接龙、不会对话”的工程难题，系统会给 Prompt “加料”。例如将你的提问包装成 `使用者问：[Prompt] AI回答：`。这种结构强制模型将概率分布收敛在“回答”而非“延续问题”上

# RAG(**检索增强生成**)

- ** 引入外部约束**：RAG 的核心逻辑不是改变模型的生成机制，而是通过**干预输入序列（Prompt）**来强行坍缩概率分布的空间
- 有了 RAG，模型变成了根据给定的参考资料进行“归纳总结”
