# 晶闸管（SCR）以及衍生的 TRIAC 与 GTO

> [!abstract] 核心本质
> SCR 是半控型功率器件：门极能触发它导通，但普通 SCR 不能靠 MCU 拉低门极来关断。它真正关断依赖主回路电流降到维持电流以下，或外部换流条件成立。

## 核心结论

晶闸管适合高压大电流、低频或交流过零相关控制；嵌入式控制它时，重点不是持续输出高电平，而是给出足够可靠的触发脉冲，并理解 $I_L$、$I_H$、$di/dt$ 和 $du/dt$。

## 什么时候用

- 交流调光、交流调压、可控整流、软启动、固态开关。
- 需要高压大电流能力，但控制频率不高。
- 有过零检测、相位控制或自然换流条件的交流系统。
- MCU 通过光耦隔离触发低压侧逻辑，高压侧由 SCR/TRIAC 承担功率。

## 什么时候不用

- 直流负载没有强迫换流设计，却希望 MCU 随时关断。
- 高频 [[TIM定时器基础概念|PWM]] 或快速电机控制，通常优先 [[MOSFET]] / [[IGBT]]。
- 感性交流负载没有处理换流失败、$dv/dt$ 误触发和吸收回路。
- 不了解擎住电流 $I_L$ 和维持电流 $I_H$，只按 GPIO 高低电平想象控制。

## 控制本质

SCR 是四层 PNPN 结构，有三个端子：

| 端子 | 英文 | 作用 |
|---|---|---|
| A | Anode | 阳极，主电流流入端 |
| K | Cathode | 阴极，主电流流出端 |
| G | Gate | 门极，触发导通端 |

它可以等效成相互正反馈耦合的 PNP 和 NPN 两个三极管。门极注入电流后，内部正反馈建立，器件从阻断进入导通。

```mermaid
flowchart LR
    Gate["门极触发电流 $I_G$"] --> Regen["内部正反馈增强"]
    Regen --> Conduct["SCR 导通"]
    Conduct --> Latch["阳极电流超过 $I_L$ 后锁存"]
    Latch --> Hold["只要 $I_A > I_H$ 就维持导通"]
```

普通 SCR 一旦导通，门极信号撤掉也会继续导通。只有当阳极电流降到维持电流 $I_H$ 以下，或者外部电路让它反向偏置，它才恢复阻断。

## 关键参数

| 参数 | 含义 | 嵌入式关注点 |
|---|---|---|
| $V_{DRM}$ / $V_{RRM}$ | 正/反向重复峰值电压 | 覆盖交流峰值、浪涌和裕量 |
| $I_T$ | 通态电流 | 结合散热和浪涌能力看 |
| $I_{GT}$ | 门极触发电流 | 驱动脉冲必须可靠超过它 |
| $I_L$ | 擎住电流 | 触发后主电流必须超过它才能锁存 |
| $I_H$ | 维持电流 | 主电流低于它才会关断 |
| $di/dt$ | 通态电流上升率 | 太大会局部过热 |
| $du/dt$ | 阻断态电压上升率 | 太大会误触发 |

## 损耗与热

晶闸管导通后有约 1V 到 2V 量级通态压降，大电流下 [[导通损耗]] 依然很可观。它通常不用于高频硬开关，因为开关过程、恢复时间和换流条件都比较苛刻。

SCR 开通不是整个芯片同时导通，而是从门极附近逐步扩展。如果外部电流上升太快，电流会集中在局部区域，引发热点。

关断后器件还需要恢复阻断能力。如果阳极-阴极电压上升太快，内部结电容会产生位移电流，可能造成误触发。

## 驱动与保护

- 门极触发要有足够电流、足够宽度和隔离裕量。
- 交流控制常配合过零检测和定时器延迟触发。
- 感性负载需要 RC 吸收、压敏电阻或其他 $du/dt$ 抑制方案。
- 大功率 SCR 常需要串联电感或进线电抗器限制 $di/dt$。
- 直流 SCR 若要关断，必须设计外部换流路径。

TRIAC 可以粗略理解为双向晶闸管，适合交流负载控制。GTO 是可关断晶闸管，但关断需要较大的反向门极电流，驱动难度高，很多现代场景会改用 IGBT/IGCT 等器件。

## 调试波形

用 [[../../示波器/01-原理与认知/1.1-示波器是什么|示波器]] 优先看：

- 门极触发脉冲：幅度、宽度、相位是否足够。
- 阳极电流：触发后是否超过 $I_L$，过零后是否低于 $I_H$。
- $U_{AK}$：阻断态尖峰、导通压降和换流过程。
- 交流相位：过零检测、触发角和负载电流相位差。
- 感性负载：是否存在 TRIAC 换流失败或误触发。

## 常见误区

- 以为门极拉低就能让普通 SCR 关断。
- 触发脉冲太短，没有保证阳极电流超过 $I_L$。
- 直流电路中使用 SCR 却没有设计强迫换流。
- 交流感性负载用 TRIAC 时忽略换流失败和误导通。
- 没有处理 $du/dt$，导致器件无命令导通。
