> [!abstract] 核心本质
> TIM 不仅是简单的“计数器”，而是嵌入式系统的**心跳引擎**与**时序控制器**。它通过硬件级的时间基准生成、波形输出与信号捕获，解决了 CPU 在精确时序控制上的效率瓶颈，是实现 RTOS 调度、电机控制（FOC）及通信协议解析的核心外设。

## 1. 核心定义

TIM（Timer）是一种集成在 MCU 内部的专用硬件计数器模块。其核心价值在于**将 CPU 从繁琐的延时循环和周期性轮询中解放出来**，通过硬件自动完成时基维护、PWM 生成及外部信号测量，实现“零 CPU 占用”的精确时序控制。

## 2. 底层原理

### 2.1 时基单元
STM32 定时器的核心由三个 16/32 位寄存器构成“时基三剑客”，决定了定时器的频率与周期。

| 寄存器 | 名称 | 核心功能 | 关键逻辑 |
| :--- | :--- | :--- | :--- |
| **PSC** | 预分频器 | 对系统时钟进行降频 | 实际分频系数 = $PSC + 1$ |
| **CNT** | 计数器 | 实时计数值 | 支持向上/向下/中心对齐计数 |
| **ARR** | 自动重装载寄存器 | 决定计数周期（溢出阈值） | 周期 = $ARR + 1$ |

**频率计算公式**：
$$ f_{out} = \frac{f_{timer\_clock}}{(PSC + 1) \times (ARR + 1)} $$

### 2.2 影子寄存器机制
> [!tip] 硬件设计的精髓
> 用户写入 `ARR`/`PSC` 的值首先存入**预装载寄存器**，仅在发生 Update Event（溢出）时，硬件才将其同步至**影子寄存器**生效。
> 
> - **作用**：防止动态修改参数时产生波形毛刺或截断。
> - **配置**：CubeMX 中务必开启 `Auto-reload preload = Enable`。

### 2.3 时钟树与总线架构
TIM 的时钟源并非固定，取决于其挂载的总线位置（以 STM32F407 为例）：

- **APB2 总线（高速）**：挂载 TIM1, TIM8, TIM9~TIM11。
    - 时钟频率：$HCLK / 2 \times 2 = 168MHz$（当 APB2 分频系数非 1 时，硬件自动 x2）。
- **APB1 总线（低速）**：挂载 TIM2~TIM5, TIM6, TIM7, TIM12~TIM14。
    - 时钟频率：$HCLK / 4 \times 2 = 84MHz$（硬件自动 x2）。

> [!danger] 致命陷阱：APB 倍频机制
> 若 APB 预分频系数非 1，定时器时钟会自动倍频。**切勿直接使用总线频率计算**，务必在 CubeMX 的 `Clock Configuration` 界面查看 `APBx Timer clocks` 的实际数值。

## 3. 硬件分类与选型

STM32 定时器按功能复杂度划分为四个阶级，选型需“好钢用在刀刃上”。

| 类型 | 代表型号 | 总线 | 位宽 | 核心特性与典型应用 |
| :--- | :--- | :--- | :--- | :--- |
| **高级定时器** | TIM1, TIM8 | APB2 | 16位 | **电机控制专用**。集成死区插入、互补输出、刹车输入、重复计数器 (RCR)。用于 BLDC/FOC 驱动。 |
| **32位通用定时器** | TIM2, TIM5 | APB1 | **32位** | **长周期测量**。用于编码器测速（防溢出）、高精度时间戳。可连续计数数小时不溢出。 |
| **16位通用定时器** | TIM3, TIM4... | APB1/APB2 | 16位 | **日常杂活**。普通 PWM 生成、按键消抖、系统 Tick。无死区/刹车功能。 |
| **基本定时器** | TIM6, TIM7 | APB1 | 16位 | **内部专用**。无外部引脚，仅用于触发 DAC 转换或作为 RTOS 的纯净时基。 |

## 4. 实战应用落地

### 4.1 场景一：周期性中断与 PWM 输出
**需求**：TIM3 产生 1ms 中断，并输出 10kHz/50% PWM。
**配置逻辑**：
1.  **时钟源**：Internal Clock (假设 84MHz)。
2.  **时基配置**：
    - `PSC` = 83 (84MHz / 84 = 1MHz, 1us/step)。
    - `ARR` = 99 (100us 周期 -> 10kHz)。
3.  **PWM 配置**：
    - `Pulse (CCR)` = 50 (50% 占空比)。

**代码实现**：
```c
/* main.c 启动逻辑 */
// 清除初始化时由 UG 位产生的无效中断标志
__HAL_TIM_CLEAR_FLAG(&htim3, TIM_FLAG_UPDATE); 
HAL_TIM_Base_Start_IT(&htim3); // 启动定时器中断
HAL_TIM_PWM_Start(&htim3, TIM_CHANNEL_1); // 启动 PWM

/* 中断回调函数 */
void HAL_TIM_PeriodElapsedCallback(TIM_HandleTypeDef *htim) {
    if (htim->Instance == TIM3) {
        // 业务逻辑：务必短小精悍，禁用 delay/printf
        static uint8_t cnt = 0;
        if (++cnt >= 10) { // 10ms 翻转一次 LED
            HAL_GPIO_TogglePin(LED_GPIO_Port, LED_Pin);
            cnt = 0;
        }
    }
}
```

### 4.2 场景二：编码器接口
**痛点**：使用外部中断（EXTI）处理编码器信号会导致 CPU 过载。
**方案**：配置 TIM 为 Encoder Mode。
- **原理**：硬件自动识别 A/B 相相位差，CNT 自动增减，支持 4 倍频。
- **优势**：**CPU 零参与**，仅需在任务中读取 `CNT` 寄存器。
- **选型建议**：优先使用 **32位 TIM2/TIM5**，彻底解决高速运转时的 16 位溢出问题。

### 4.3 场景三：硬件级联动
**架构原则**：能用硬件联动，绝不用软件中断。
- **TIM -> ADC**：配置 TIM 的 TRGO 信号直接触发 ADC 采样，配合 DMA 搬运数据。实现“定时采样 - DMA 存储 - CPU 仅处理结果”的零开销闭环。
- **TIM -> DMA (Burst)**：利用 DMA 突发传输，在定时器溢出时自动更新多个 CCR 寄存器，实现复杂波形（如 SPWM）输出，释放 CPU。

## 5. 避坑与边界条件

> [!warning] 避坑指南：初始化瞬间的中断陷阱
> **现象**：调用 `HAL_TIM_Base_Start_IT` 后立即进入一次中断。
> **根因**：`HAL_Init` 内部通过置位 UG 位更新寄存器，导致 UIF 标志被置位。
> **解法**：在开启中断前，显式调用 `__HAL_TIM_CLEAR_FLAG(&htimx, TIM_FLAG_UPDATE);`。

> [!warning] 避坑指南：动态修改参数的毛刺
> **场景**：闭环控制中频繁修改 `ARR` 或 `CCR`。
> **风险**：若关闭 Preload，新值立即生效。若 `CNT > 新ARR`，定时器将计数至满量程才溢出，导致周期异常。
> **解法**：始终开启 `Auto-reload preload`。

> [!warning] 避坑指南：PCB Layout 干扰
> **场景**：高级定时器 TIM8 驱动电机时，I2C 传感器频繁死机。
> **根因**：高频 PWM 信号产生强 EMI 干扰。
> **解法**：PWM 走线远离敏感模拟信号（ADC/I2C），必要时做包地处理。
