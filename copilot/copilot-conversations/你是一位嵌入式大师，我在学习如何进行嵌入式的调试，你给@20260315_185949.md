## 一、调试前置：可观测性设计

调试的本质是**信息获取**。如果你没有预留观测点，神仙也难调。

### 1.1 日志系统（软件观测窗口）

```c
// 推荐的分级日志宏，生产环境可裁剪
#define LOG_LEVEL_ERROR  1
#define LOG_LEVEL_WARN   2
#define LOG_LEVEL_INFO   3
#define LOG_LEVEL_DEBUG  4
#define LOG_LEVEL_TRACE  5

#ifndef LOG_LEVEL
#define LOG_LEVEL LOG_LEVEL_INFO  // 发布时调低
#endif

#define LOG_E(fmt, ...)  do { if(LOG_LEVEL >= LOG_LEVEL_ERROR)  \
    printf("[E][%s:%d] " fmt "\r\n", __func__, __LINE__, ##__VA_ARGS__); } while(0)
#define LOG_W(fmt, ...)  do { if(LOG_LEVEL >= LOG_LEVEL_WARN)   \
    printf("[W][%s:%d] " fmt "\r\n", __func__, __LINE__, ##__VA_ARGS__); } while(0)
#define LOG_I(fmt, ...)  do { if(LOG_LEVEL >= LOG_LEVEL_INFO)   \
    printf("[I] " fmt "\r\n", ##__VA_ARGS__); } while(0)

// 关键：带时间戳的日志，用于时序分析
#define LOG_T(fmt, ...)  do { if(LOG_LEVEL >= LOG_LEVEL_TRACE) \
    printf("[T][%lu] " fmt "\r\n", HAL_GetTick(), ##__VA_ARGS__); } while(0)
```

**避坑**：
- 高频中断（如PWM中断）内禁用 `printf`，会阻塞CPU导致时序崩溃
- 改用环形缓冲区暂存，主循环输出

## 二、软件调试流程

### 2.1 GDB 调试核心命令

| 命令 | 作用 | 工程场景 |
|---|---|
| `break func_name` | 函数入口断点 | 追踪调用链 |
| `break file.c:123 if var>10` | 条件断点 | 定位数据异常 |
| `watch variable` | 数据断点 | 捕获变量被谁修改 |
| `info registers` | 查看寄存器 | HardFault 分析 |
| `x/10xw 0x20000000` | 查看内存 | 检查栈/堆状态 |
| `backtrace` | 调用栈回溯 | 定位崩溃位置 |
| `thread info` | 查看RTOS任务 | 多任务死锁排查 |

### 2.2 HardFault 排查标准流程

```c
// 在 HardFault_Handler 中添加：
void HardFault_Handler(void) {
    volatile uint32_t *stack_ptr;
    __asm volatile ("TST LR, #4 \n"
                    "ITE EQ \n"
                    "MRSEQ %0, MSP \n"
                    "MRSNE %0, PSP \n" : "=r"(stack_ptr));
    
    printf("HardFault!\n");
    printf("R0  = 0x%08X\n", stack_ptr[0]);
    printf("R1  = 0x%08X\n", stack_ptr[1]);
    printf("R2  = 0x%08X\n", stack_ptr[2]);
    printf("R3  = 0x%08X\n", stack_ptr[3]);
    printf("R12 = 0x%08X\n", stack_ptr[4]);
    printf("LR  = 0x%08X\n", stack_ptr[5]);  // 返回地址
    printf("PC  = 0x%08X\n", stack_ptr[6]);  // 崩溃地址
    printf("xPSR= 0x%08X\n", stack_ptr[7]);
    
    // 检查故障状态寄存器
    printf("CFSR= 0x%08X\n", SCB->CFSR);
    printf("HFSR= 0x%08X\n", SCB->HFSR);
    printf("MMFAR=0x%08X\n", SCB->MMFAR);
    printf("BFAR= 0x%08X\n", SCB->BFAR);
    
    while(1);
}
```

**CFSR 解读速查**：
- `BFARVALID` (bit 15) = 1 → 总线错误，看 BFAR 地址
- `PRECISERR` (bit 10) = 1 → 数据访问越界
- `UNALIGNED` (bit 8) = 1 → 非对齐访问
- `DIVBYZERO` (bit 9) = 1 → 除零错误

### 2.3 内存问题排查

```c
// 栈溢出检测：在启动文件中填充特定模式
// startup.s 中：
// Stack_Mem       EQU     0x20000000
// 在 main 开头检查：
bool check_stack_overflow(void) {
    extern uint32_t _estack;
    extern uint32_t _Min_Stack_Size;
    uint32_t *stack_base = &_estack - ((uint32_t)&_Min_Stack_Size / 4);
    
    // 如果栈底被改写，说明溢出
    if (*stack_base != 0xDEADBEEF) {  // 假设初始化时填充
        LOG_E("Stack overflow detected!");
        return true;
    }
    return false;
}

// 堆碎片检测（如果用 malloc）
#ifdef USE_DYNAMIC_MEMORY
void check_heap_fragmentation(void) {
    size_t free = xPortGetFreeHeapSize();
    size_t max_block = xPortGetMinimumEverFreeHeapSize();
    LOG_I("Heap: free=%u, max_block=%u, frag=%d%%", 
          free, max_block, 100 - (max_block * 100 / free));
}
#endif
```

## 三、硬件调试流程

### 3.1 必备工具清单

| 工具 | 核心用途 | 关键指标 |
|---|---|
| 示波器 | 时序、信号质量 | 带宽≥100MHz，4通道 |
| 逻辑分析仪 | 协议解码（I2C/SPI/UART） | 采样率≥100MSa/s |
| 万用表 | 电源、通断、基本电平 | - |
| USB-TTL | 串口日志 | - |
| ST-Link/J-Link | 在线调试、烧录 | SWD接口 |

### 3.2 示波器调试要点

**电源纹波测量**：
- 探头接地线要短（用接地弹簧），否则引入噪声
- 测量点尽量靠近芯片引脚
- 纹波要求：通常 < 50mV（数字电路）

**信号完整性检查**：
```
观察项目：
1. 上升/下降沿是否陡峭（< 10ns 为佳）
2. 是否有过冲/振铃（> VDD+0.3V 可能损坏芯片）
3. 电平是否达标（VIH/VIL 阈值）
```

**I2C 时序检查**：
- START 条件：SCL 高时 SDA 下降沿
- STOP 条件：SCL 高时 SDA 上升沿
- 数据建立时间：SDA 变化后 SCL 上升沿前的时间

### 3.3 逻辑分析仪协议解码

```
连接方式：
1. 先接 GND（共地）
2. 再接信号线（SCL/SDA/MOSI/MISO/RX/TX）
3. 设置采样率：至少 10 倍于信号频率

常见问题：
- I2C 无 ACK → 检查上拉电阻（2.2K~10K）、从机地址、供电
- SPI 数据错位 → 检查 CPOL/CPHA 配置、CS 片选时序
- UART 乱码 → 检查波特率误差（< 2%）、时钟源精度
```

## 四、系统级调试流程

### 4.1 调试决策树

```
问题现象
    │
    ├─ 完全无反应 ──→ 检查电源、时钟、复位
    │
    ├─ 运行后死机 ──→ HardFault 分析、栈溢出检查
    │
    ├─ 外设不工作 ──→ 时钟使能 → GPIO配置 → 外设初始化 → 中断配置
    │
    ├─ 通信失败 ──→ 示波器看波形 → 逻辑分析仪解码 → 检查时序参数
    │
    └─ 间歇性故障 ──→ 竞态条件、中断嵌套、电源纹波、EMI干扰
```

### 4.3 中断相关问题

```c
// 中断优先级配置原则：
// 1. 抢占优先级：数值越小越优先
// 2. FreeRTOS 宏 configMAX_SYSCALL_INTERRUPT_PRIORITY 以下的优先级才能调用 API
// 3. 高频中断（如电机PWM）优先级要高，但不要调用任何 OS API

// 错误示例：在中断中调用可能阻塞的函数
void EXTI0_IRQHandler(void) {
    // xQueueSend 可能阻塞！应该用 FromISR 版本
    xQueueSend(myQueue, &data, portMAX_DELAY);  // ❌ 错误！
    
    // 正确做法：
    BaseType_t xHigherPriorityTaskWoken = pdFALSE;
    xQueueSendFromISR(myQueue, &data, &xHigherPriorityTaskWoken);  // ✅
    portYIELD_FROM_ISR(xHigherPriorityTaskWoken);
}
```

## 六、大师的工程建议

### 6.1 调试心态

1. **假设驱动**：先提出假设，再设计实验验证，不要盲目试错
2. **控制变量**：每次只改一个地方，观察结果变化
3. **保留现场**：崩溃时第一时间保存寄存器、栈、变量状态
4. **文档记录**：Bug 修复后记录根因，建立知识库

### 6.2 预防性设计

```c
// 1. 断言检查（开发阶段启用）
#define ASSERT(expr)  do { if(!(expr)) { \
    LOG_E("Assert failed: %s, %s:%d", #expr, __FILE__, __LINE__); \
    while(1); }} while(0)

// 2. 参数校验
void motor_set_speed(int16_t speed) {
    ASSERT(speed >= -MOTOR_MAX_SPEED && speed <= MOTOR_MAX_SPEED);
    // ...
}

// 3. 状态机防护
typedef enum { STATE_IDLE, STATE_RUNNING, STATE_ERROR } MotorState;
void motor_update(MotorState *state, Event_t event) {
    switch(*state) {
        case STATE_IDLE:
            if (event == EVT_START) *state = STATE_RUNNING;
            else LOG_W("Invalid event %d in IDLE", event);
            break;
        // ...
    }
}

// 4. 看门狗喂狗策略
// - 主循环喂狗，检测死循环
// - 关键任务超时也触发复位
```

### 6.3 调试工具箱速查

| 问题类型 | 首选工具 | 次选工具 |
|---|---|
| 程序崩溃 | GDB + HardFault分析 | 日志回溯 |
| 内存泄漏 | Map文件 + 栈检查 | 静态分析工具 |
| 时序问题 | 示波器 + GPIO探针 | 逻辑分析仪 |
| 通信协议 | 逻辑分析仪 | 示波器 |
| 电源问题 | 示波器（AC耦合） | 万用表 |
| EMI干扰 | 近场探头 | 屏蔽实验 |
