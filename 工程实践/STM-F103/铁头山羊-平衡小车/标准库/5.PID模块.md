# PID 模块：从电机建模到平衡车调参

> [!abstract] 实战场景
> 这篇笔记不是为了把控制理论完整讲一遍，而是为了回答一个工程问题：平衡小车已经有了 [[2.电机模块]]、[[3.编码器模块]] 和 [[4.MPU6050]]，为什么还需要 PID，PID 应该接在哪里，参数又该怎么调。
>
> 对这个项目来说，PID 是把“姿态误差/速度误差”翻译成“电机输出”的中间层。它上接 [[6.倒立摆]] 的姿态目标和 [[7.速度环]] 的速度目标，下接电机 PWM 输出。

> [!note] 快速结论
> - PID 的工程本质：用误差驱动执行器，让被控对象的输出跟踪设定值。
> - 平衡车里至少会遇到两个控制问题：角度站稳，以及速度不漂；所以后面自然会走向串级 PID。
> - 写代码时先把 PID 做成独立模块，再接入 motor 层；调参时先限幅、再看震荡、最后处理积分和微分的副作用。
> - 截图里的推导不要死背，真正要留下的是“从模型理解参数、从现象反推问题”的工程链路。

## 总体主线

![](assets/5.PID模块/{FE8BB56C-D434-4452-BEBF-2DEC44C2B233}.png)

**图意：** 这张图把课程路线分成三段：先认识控制系统指标，再建立电机模型，最后用 Bode 图和三段论辅助 PID 参数整定。

**工程结论：** 学 PID 不是先背公式，而是先明确闭环里的每个角色：目标值、反馈值、误差、控制器、执行器、被控对象。这个角色链一旦清楚，后面才知道 PID 输出应该给谁、反馈应该从哪里来。

## PID 认知

![](assets/5.PID模块/{25F77DA5-F2DF-42D6-8573-8E584CBBFBA7}.png)

**图意：** 这张图强调 PID 参数调试的方向感：`Kp` 主要影响响应强度，`Ki` 消除静差但容易积累副作用，`Kd` 抑制变化趋势但怕噪声。

**工程结论：**
- `P` 负责“现在误差有多大”，太小站不住，太大容易震。
- `I` 负责“过去欠了多少账”，能消除稳态误差，但必须做积分限幅。
- `D` 负责“误差变化有多快”，能提前刹车，但传感器噪声大时会把输出打得很毛。

> [!warning] 易错点：PID 不是万能参数表
> PID 参数必须绑定对象、采样周期、输出限幅和反馈噪声来看。同一套参数换了电机、电池电压、轮子、采样频率，表现都可能完全变样。

## 控制系统主线

- **设定值：** 你希望系统达到的目标，例如小车角度为 0 度、速度为 0 或某个目标速度。
- **反馈值：** 传感器测到的实际状态，例如 [[4.MPU6050]] 的姿态角、[[3.编码器模块]] 的轮速。
- **误差：** `error = target - feedback`，PID 只认识误差。
- **控制器：** PID 根据误差计算输出。
- **执行器：** [[2.电机模块]] 根据 PID 输出改变 PWM 和电机方向。
- **被控对象：** 电机、车轮、车体、地面摩擦共同组成真实系统。

![](assets/5.PID模块/{7752BD79-C425-47A2-AC91-61DC0A869A3F}.png)

**图意：** 控制系统的基本闭环结构：控制器不是直接“控制结果”，而是通过执行器改变被控对象，再由反馈通道把结果带回来。

**工程结论：** 写代码时不要让 PID 模块直接读取所有硬件。更清晰的接口是：外部传入 `target` 和 `measurement`，PID 只计算 `output`。

![](assets/5.PID模块/{0B10A9C6-1459-4669-BB21-F352F47EFE20}.png)

**图意：** 控制系统常用指标通常围绕“稳、准、快”展开，例如超调、调节时间、稳态误差。

**工程结论：**
- “快”不等于好。小车响应太快，可能表现为来回抽动。
- “准”不等于稳。误差看起来变小，但如果输出长期顶到限幅，系统已经失真。
- “稳”是第一优先级。平衡车先站住，再谈速度和手感。

## 电机建模

> [!note] 思考过程
> 这一段原来最容易写成“公式截图合集”。真正有用的主线是：电压产生电流，电流产生电磁转矩，转矩克服负载和惯量，最后表现成角速度和车体姿态变化。

![](assets/5.PID模块/{400AAB03-62B6-4D50-93FE-543613C275D8}.png)

**图意：** 建模入口：从电机的电学方程和机械方程出发，把真实电机抽象成可以分析的系统。

**工程结论：** 建模不是为了把所有参数精确测到，而是为了知道哪些量会影响调参：电阻、电感、反电动势、转矩常数、负载惯量、摩擦。

### 电学方程

![](assets/5.PID模块/{7AEF47F4-7B7D-46AB-BEE1-D98FF0A973FC}.png)

**图意：** 电机电枢回路可以看成电阻、电感、反电动势组成的动态系统。

**工程结论：** PWM 占空比不是“速度命令”本身，它只是改变等效输入电压。电机最终速度还会被负载、反电动势、电池电压和摩擦影响。

### 刚体运动和力矩

![](assets/5.PID模块/{EFD1B5D4-161F-4C85-9273-3DA73919F362}.png)

**图意：** 把电磁力矩接到机械转动方程，解释为什么电流变化最终会变成轮子角速度变化。

**工程结论：** 对平衡车来说，电机不是孤立对象。车轮半径、车体重心、地面摩擦都会改变“同样 PWM 输出”的实际效果。

> [!warning] 修正：转动惯量不能一律写成 `J = M * R^2`
> `J = M * R^2` 只适合质点或薄环这类简化模型。更通用的理解是 `J = sum(m_i * r_i^2)`，质量离转轴越远，越难被加速。工程上不一定要精确推导，但要知道惯量会直接影响响应速度和超调。

![](assets/5.PID模块/{8D84B89D-B58A-4315-88FA-57E2D6DA55FE}.png)

**图意：** 转动惯量的测量或估算方法。

**工程结论：** 如果模型参数不准，PID 仍然能工作，但调参会变成纯手感。参数测量的价值是缩小调参搜索范围。

![](assets/5.PID模块/{2B4463B2-3D41-4F8F-8315-DB1A86943847}.png)

**图意：** 力矩关系汇总，把电机输出、负载和运动状态放到同一个机械模型里。

**工程结论：** 当小车“有输出但动不起来”时，不只查 PID，也要查摩擦、死区、电池电压、驱动器压降和电机方向。

![](assets/5.PID模块/{3DD30AEB-7CEB-40BE-BF78-20CDE0705FBB}.png)

**图意：** 电学模型和机械模型合成后的总体结构。

**工程结论：** 这是从“给电压”到“得到速度/位置”的因果链。PID 调参时，`Kp`、`Ki`、`Kd` 实际是在改变这条因果链的闭环特性。

## 工程实现

![](assets/5.PID模块/{91EDEEC8-9965-47B5-8154-EA3C336BB064}.png)

**图意：** PID 代码落地的整体布局：结构体、初始化、计算函数、接入电机层、测试。

**工程结论：** PID 模块应该只保存自己的状态，不直接依赖某个具体外设。这样同一个模块才能复用到角度环、速度环甚至后续方向控制。

### 结构体建议

```c
typedef struct {
    float kp;
    float ki;
    float kd;

    float target;
    float measurement;
    float error;
    float last_error;

    float integral;
    float output;

    float output_limit;
    float integral_limit;
    float dt;
    uint8_t first_run;
} pid_t;
```

**设计原则：**
- `dt` 必须明确，积分和微分都依赖采样周期。
- `output_limit` 对应电池、电机驱动和 PWM 的实际能力。
- `integral_limit` 用来防止积分饱和。
- `first_run` 用来避免第一次计算时微分项突然爆掉。

### `app_pid_compute`

![](assets/5.PID模块/{1C0C17A0-19B5-4570-92AB-529172C0E191}.png)

**图意：** PID 计算函数的核心：比例、积分、微分分别计算，再合成输出。

**工程结论：** 积分是误差面积，微分是误差斜率；这两个量最容易被采样周期和噪声影响。

```c
float app_pid_compute(pid_t *pid, float target, float measurement)
{
    pid->target = target;
    pid->measurement = measurement;
    pid->error = pid->target - pid->measurement;

    if (pid->first_run) {
        pid->last_error = pid->error;
        pid->first_run = 0;
    }

    pid->integral += pid->error * pid->dt;
    pid->integral = limit(pid->integral, -pid->integral_limit, pid->integral_limit);

    float derivative = (pid->error - pid->last_error) / pid->dt;
    pid->output = pid->kp * pid->error
                + pid->ki * pid->integral
                + pid->kd * derivative;

    pid->output = limit(pid->output, -pid->output_limit, pid->output_limit);
    pid->last_error = pid->error;
    return pid->output;
}
```

> [!warning] 修正：输出限幅不是“理论电压范围”而是执行器约束
> 原文提到限制在 `-8.4V ~ 8.4V`，这个范围要理解为电池和驱动允许的等效输出边界。真正写到代码里，通常会映射成 PWM 占空比或电机驱动命令，而不是直接输出电压。

![](assets/5.PID模块/{ACB4B4CA-F23B-49F8-895D-AAC1012F735A}.png)

**图意：** 工程接入和测试过程。

**工程结论：** 每次接入新环节都要单独测试。先测 PID 纯函数，再测 motor 输出，再测整车闭环。否则最后会变成“哪哪都可能错”。

## PID 改进思路

### 输出限幅

![](assets/5.PID模块/{E6C681EE-B312-419F-A1DB-9F2A881157C2}.png)

**图意：** 给 PID 输出增加限幅。

**工程结论：** 输出限幅保护电机和驱动器，也让控制器符合真实执行器能力。没有限幅的 PID 在仿真里可能很漂亮，在车上可能直接把输出打满。

### 积分限幅

![](assets/5.PID模块/{CBF877F6-FD31-4B43-9ACE-4D8AAF7844F4}.png)

**图意：** 给积分项增加限幅。

**工程结论：** 积分限幅是为了防止长时间误差把积分项堆到很大。典型现象是车已经过目标点了，但控制器还在用过去累积的误差继续推。

### PID 复位和首次运行处理

![](assets/5.PID模块/{F902BBA3-AEDD-4E39-A31C-9F1FF8C4BD62}.png)

**图意：** 首次运行时忽略或保护积分、微分项，复位时清空暂存状态。

**工程结论：**
- 切换模式、停车、重新使能控制器时，要清空 `integral` 和 `last_error`。
- 第一次运行不要直接计算微分，否则 `last_error` 的旧值会制造一次假的大输出。

## 调参和排错

| 现象 | 优先怀疑 | 修正动作 |
| --- | --- | --- |
| 小车完全不动 | 电机方向、PWM 映射、死区、输出限幅太小 | 先脱离 PID 测 [[2.电机模块]] |
| 一上电就猛冲 | 反馈方向反了、目标值单位错、`Kp` 太大 | 检查误差符号和单位 |
| 来回快速抖动 | `Kp` 太大、`Kd` 放大噪声、采样周期不稳 | 降 `Kp`，过滤反馈，固定 `dt` |
| 慢慢漂走 | 缺少速度环、积分不足、编码器反馈未接入 | 接入 [[7.速度环]] 或检查 [[3.编码器模块]] |
| 越调越怪 | 输出长期限幅、积分饱和、复位状态没清 | 加限幅，调试时记录 PID 内部状态 |

> [!tip] 复盘法
> 每次调参不要只记录“参数是多少”，还要记录现象：是否超调、是否震荡、是否能回正、输出是否打满、传感器是否抖。真正有用的经验会沉淀到 [[9.整体的工程思考和错误问题]]。
