# ADC 采样电源：从定时触发到电池电压换算

> [!abstract] 实战场景
> 这篇笔记解决的是平衡小车的“电池电压怎么稳定采样、怎么换算、怎么用 LED 表示电量”的问题。
>
> 工程链路是：电池电压经过分压电路进入 `PB0 / ADC1_CH8`，由 `TIM3 TRGO` 每 10 ms 触发一次 ADC 转换，转换完成后进入 `EOC` 中断读取原始值，再换算成真实电池电压，最后用于电量显示或低压保护。

> [!note] 快速结论
> - ADC 驱动的重点不是“背初始化函数”，而是把模拟输入、ADC 时钟、触发源、采样时间、数据读取、物理量换算串成闭环。
> - 本项目是单通道、规则组、定时器触发、EOC 中断读取；还没有引入扫描模式、注入组、DMA 或 RTOS。
> - `ADC_DataAlign_Right` 右对齐是最常用方式，12 位结果直接落在 `uint16_t` 低 12 位，后续换算更直观。
> - 电池电压采样最好先通过 USART 打印验证，再接 LED 电量显示；不要一开始就把采样、显示、控制全揉在一起。

## 总体主线

```text
电池电压
  -> 电阻分压
  -> PB0 / ADC1_CH8
  -> TIM3 每 10 ms 产生 TRGO
  -> ADC1 规则组转换
  -> EOC 中断读取 DR
  -> raw -> 电池电压
  -> USART 测试 / LED 电量显示 / 低压保护
```

**工程结论：** ADC 采样不是孤立外设。它同时牵涉 GPIO、RCC、ADC、TIM、NVIC、USART 测试，以及后续裸机周期任务调度。

> [!warning] 修正：原文里的 “TCB 结构体”
> 这里更准确的说法是 `InitTypeDef` 配置结构体，例如 `GPIO_InitTypeDef`、`ADC_InitTypeDef`、`TIM_TimeBaseInitTypeDef`。它们不是外设本身，只是标准库用来批量写寄存器的配置载体。

## ADC 硬件链路

![](assets/1.ADC-采样电源/file-20260504005617421.png)

**图意：** 左侧是 STM32F103 的时钟树，右侧是 ADC 通道、规则组/注入组、外部触发、EOC/JEOC 中断和数据寄存器结构。

**工程结论：**
- `PB0` 对应 `ADC1_Channel_8`，本项目用它采集电池分压后的模拟电压。
- `ADC1` 时钟来自 `PCLK2` 分频，STM32F1 常见约束是 ADC 时钟不超过 14 MHz；`72 MHz / 6 = 12 MHz` 是合理选择。
- 本项目用规则组和 `EOC` 中断；图里同时出现的注入组和 `JEOC` 是另一套路径，不要混在一起理解。
- 如果以后要多通道采样、连续采样或高频采样，再考虑扫描模式和 DMA。

## ADC 初始化

> [!note] 初始化思路
> 外设初始化可以按“先把信号脚接进来，再把外设时钟和模式配好，最后打开中断和总开关”的顺序记。不要一上来就抄函数，先在脑子里过一遍数据流。

```c
static void ADC1_Battery_Init(void)
{
    RCC_APB2PeriphClockCmd(RCC_APB2Periph_GPIOB, ENABLE);
    RCC_APB2PeriphClockCmd(RCC_APB2Periph_ADC1, ENABLE);

    GPIO_InitTypeDef gpio = {0};
    gpio.GPIO_Pin = GPIO_Pin_0;
    gpio.GPIO_Mode = GPIO_Mode_AIN;
    GPIO_Init(GPIOB, &gpio);

    RCC_ADCCLKConfig(RCC_PCLK2_Div6);  // 72 MHz / 6 = 12 MHz

    ADC_InitTypeDef adc = {0};
    adc.ADC_Mode = ADC_Mode_Independent;
    adc.ADC_ScanConvMode = DISABLE;
    adc.ADC_ContinuousConvMode = DISABLE;
    adc.ADC_ExternalTrigConv = ADC_ExternalTrigConv_T3_TRGO;
    adc.ADC_DataAlign = ADC_DataAlign_Right;
    adc.ADC_NbrOfChannel = 1;
    ADC_Init(ADC1, &adc);

    ADC_RegularChannelConfig(ADC1, ADC_Channel_8, 1, ADC_SampleTime_7Cycles5);
    ADC_ExternalTrigConvCmd(ADC1, ENABLE);

    ADC_ITConfig(ADC1, ADC_IT_EOC, ENABLE);

    NVIC_InitTypeDef nvic = {0};
    nvic.NVIC_IRQChannel = ADC1_2_IRQn;
    nvic.NVIC_IRQChannelPreemptionPriority = 1;
    nvic.NVIC_IRQChannelSubPriority = 0;
    nvic.NVIC_IRQChannelCmd = ENABLE;
    NVIC_Init(&nvic);

    ADC_Cmd(ADC1, ENABLE);

    ADC_ResetCalibration(ADC1);
    while (ADC_GetResetCalibrationStatus(ADC1) == SET) {}

    ADC_StartCalibration(ADC1);
    while (ADC_GetCalibrationStatus(ADC1) == SET) {}
}
```

**工程结论：**
- `ADC_ScanConvMode = DISABLE`：单通道采样，不需要扫描。
- `ADC_ContinuousConvMode = DISABLE`：不连续采样，转换由 TIM3 触发。
- `ADC_ExternalTrigConv = ADC_ExternalTrigConv_T3_TRGO`：把采样节奏交给定时器。
- `ADC_RegularChannelConfig(..., 1, ...)`：`1` 是规则组 rank，不是“触发通道”。
- F1 标准库里建议打开 ADC 后做校准，尤其是想让电压换算更可信时。

### EOC 中断读取

```c
static volatile uint16_t g_battery_adc_raw = 0;

void ADC1_2_IRQHandler(void)
{
    if (ADC_GetITStatus(ADC1, ADC_IT_EOC) == SET) {
        g_battery_adc_raw = ADC_GetConversionValue(ADC1);
        ADC_ClearITPendingBit(ADC1, ADC_IT_EOC);
    }
}
```

> [!warning] 易错点：读数和物理量不要混在 ISR 里
> 中断里尽量只做“取走原始值、置标志位”这种短动作。电压换算、LED 显示、串口打印放到主循环周期任务里做，后面排查会轻松很多。

## TIM3 定时触发

原文的目标是每 10 ms 触发一次 ADC，这个思路是对的。更准确的表达是：`TIM3` 作为主定时器，把 update event 作为 `TRGO` 输出给 ADC。

```c
static void TIM3_TRGO_Init(void)
{
    RCC_APB1PeriphClockCmd(RCC_APB1Periph_TIM3, ENABLE);

    TIM_TimeBaseInitTypeDef tim = {0};
    tim.TIM_CounterMode = TIM_CounterMode_Up;
    tim.TIM_Prescaler = 71;
    tim.TIM_Period = 9999;
    tim.TIM_ClockDivision = TIM_CKD_DIV1;
    TIM_TimeBaseInit(TIM3, &tim);

    TIM_SelectOutputTrigger(TIM3, TIM_TRGOSource_Update);
    TIM_Cmd(TIM3, ENABLE);
}
```

**工程结论：**
- 如果 `TIM3CLK = 72 MHz`，`PSC = 71` 后计数频率为 `1 MHz`，`ARR = 9999` 后 update 周期为 `10 ms`。
- `TIM_SelectOutputTrigger(TIM3, TIM_TRGOSource_Update)` 是主模式触发输出，不是从机模式。
- `TIM_RepetitionCounter` 主要用于高级定时器，TIM3 这种通用定时器不要靠它做重复计数。
- 原文代码里 `TIM_TimeBaseInitTypeDef TIM_TimeBaseInitStruct = {0};` 写了两次，整理后去掉重复定义。

## 采样时间计算

![](assets/1.ADC-采样电源/file-20260504005617423.png)

**图意：** 这张图在估算 ADC 采样电容充电到足够精度所需的采样周期。影响因素包括信号源内阻、ADC 内部等效电阻、采样电容、ADC 位数和 ADC 时钟。

**工程结论：**
- 如果公式右侧乘了 `f_ADC`，那么算出来的是“需要多少个 ADC 时钟周期”；如果不乘 `f_ADC`，算出来的是“需要多少秒”。
- 图中示例约为 `2.79 cycle`，所以 `1.5 cycle` 不够，`7.5 cycle` 就已经满足。
- 采样时间不是越短越好。源阻抗越大，采样电容越需要更长时间充电。
- 本项目采电池电压，速度要求不高，宁可选更保守的采样时间，换稳定性。

> [!tip] 采样时间和转换时间
> STM32F1 12 位 ADC 的单次转换总时间通常可以按 `采样时间 + 12.5 cycle` 理解。比如采样时间选 `7.5 cycle`，总转换约 `20 cycle`；在 12 MHz ADC 时钟下约 `1.67 us`，远小于 10 ms 触发周期。

## 电压换算

电池是两节锂电，满电大约 `8.4 V`，但 ADC 引脚最大只能看到 `Vref` 附近的电压，通常是 `3.3 V`。所以硬件上必须先分压。

```text
电池电压
  |
 [R1]
  |
  +---- PB0 / ADC1_CH8
  |
 [R2]
  |
 GND
```

通用换算：

```c
static float battery_voltage_from_adc(uint16_t raw)
{
    const float vref = 3.3f;
    const float adc_full_scale = 4095.0f;
    const float r1 = 10000.0f;
    const float r2 = 6500.0f;

    float adc_pin_v = (raw / adc_full_scale) * vref;
    return adc_pin_v * (r1 + r2) / r2;
}
```

如果硬件分压刚好设计成“`8.4 V` 映射到 `3.3 V`”，也可以把公式压缩成：

```c
battery_v = raw / 4095.0f * 8.4f;
```

> [!warning] 易错点：`8.4 V` 不是 ADC 参考电压
> `4095` 对应的是 ADC 输入脚接近 `Vref`，不是电池直接等于 `8.4 V`。只有在分压电路把 `8.4 V` 缩放到 `3.3 V` 时，才可以写成 `raw / 4095 * 8.4`。

## USART 测试链路

![](assets/1.ADC-采样电源/file-20260504005617424.png)

**图意：** USART 由波特率发生器、发送/接收数据寄存器、状态位和中断请求组成。图里重点圈出了接收路径和 `RxNE` 相关状态。

**工程结论：** 在 ADC 模块接入 LED 或控制逻辑前，先通过 USART 打印 `raw` 和 `battery_v`，这是最便宜的验证方式。后续蓝牙通信可以继续沿用 [[8.蓝牙控制模块]] 的接收/解析思路。

```c
static void USART2_Debug_Init(void)
{
    RCC_APB2PeriphClockCmd(RCC_APB2Periph_GPIOA, ENABLE);
    RCC_APB1PeriphClockCmd(RCC_APB1Periph_USART2, ENABLE);

    GPIO_InitTypeDef gpio = {0};

    gpio.GPIO_Pin = GPIO_Pin_2;
    gpio.GPIO_Mode = GPIO_Mode_AF_PP;
    gpio.GPIO_Speed = GPIO_Speed_2MHz;
    GPIO_Init(GPIOA, &gpio);

    gpio.GPIO_Pin = GPIO_Pin_3;
    gpio.GPIO_Mode = GPIO_Mode_IPU;
    GPIO_Init(GPIOA, &gpio);

    USART_InitTypeDef usart = {0};
    usart.USART_BaudRate = 921600;
    usart.USART_WordLength = USART_WordLength_8b;
    usart.USART_StopBits = USART_StopBits_1;
    usart.USART_Parity = USART_Parity_No;
    usart.USART_Mode = USART_Mode_Tx | USART_Mode_Rx;
    USART_Init(USART2, &usart);

    USART_Cmd(USART2, ENABLE);
}
```

**工程结论：**
- `TX` 用复用推挽输出，`RX` 用上拉输入，可以避免空闲或模块断开时 RX 悬空乱跳。
- 这里的 USART 主要用于测试，不要把调试打印放进高频中断里。
- `921600` 波特率可以快速打印数据，但调试异常时也要怀疑串口工具、线材、时钟误差和 printf 阻塞。

## 裸机周期任务

原文里的重点是：不用 `delay()` 卡死 CPU，而是让主循环持续轮询，每个模块按自己的周期运行。这是裸机工程里很实用的“伪多任务”思路。

```c
while (1) {
    if (periodic_10ms()) {
        battery_sample_update();
    }

    if (periodic_100ms()) {
        battery_led_update();
    }

    if (periodic_500ms()) {
        debug_print_battery();
    }
}
```

**工程结论：**
- ADC 触发可以由 TIM3 硬件完成，主循环只处理结果。
- LED 电量显示不需要 10 ms 更新，可以放慢周期。
- USART 打印更应该降频，否则会干扰控制和采样节奏。

## 电量显示策略

| 电池电压区间 | LED 状态 | 含义 |
| --- | --- | --- |
| `< 6.0 V` | 3 灯快闪，约 100 ms | 亏电警告 |
| `6.0 V ~ 6.6 V` | 全灭 | 危险，低电量 |
| `6.6 V ~ 7.3 V` | 亮 1 颗 | 低于约 66% |
| `7.3 V ~ 8.0 V` | 亮 2 颗 | 低于约 99% |
| `>= 8.0 V` | 亮 3 颗 | 接近满电 |

> [!warning] 易错点：电压分段只是粗略电量
> 锂电电压和剩余容量不是严格线性关系，而且负载电流会造成压降。这个 LED 逻辑适合做工程提示，不适合当精确电量计。

## 调试和排错

| 现象 | 优先怀疑 | 验证动作 |
| --- | --- | --- |
| ADC 一直是 0 | GPIO 没配模拟输入、通道选错、分压点没接到 PB0 | 万用表测 PB0 电压，打印 raw |
| ADC 一直接近 4095 | 分压比错误、输入超过 Vref、地线问题 | 测 PB0 是否超过 3.3 V |
| 数值跳动很大 | 源阻抗高、采样时间太短、线太长、没有滤波 | 增大采样时间，多次平均 |
| 10 ms 触发不生效 | TIM3 TRGO 没开、ADC 外部触发没使能 | 打断点或翻转 GPIO 验证中断频率 |
| 串口打印乱码 | 波特率不一致、时钟配置不准、线序错误 | 降波特率到 115200 排查 |

> [!tip] 复盘法
> ADC 类问题先分层：先测硬件电压，再看 ADC raw，再看换算公式，最后看 LED 或控制逻辑。不要一开始就怀疑整套系统坏了。
