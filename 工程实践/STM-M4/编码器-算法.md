## 一、设计哲学

### 1.1 配置驱动

将**静态配置参数**与**动态运行状态**分离，实现一套代码适配不同硬件。

|结构体|职责|生命周期|示例字段|
|---|---|---|---|
|`encoder_cfg_t`|存放只读硬件参数|初始化后不变|极性、每脉冲位移、滤波系数|
|`encoder_wheel_sample_t`|存放运行时状态|每周期更新|累计位置、瞬时速度、滤波速度|

### 1.2 流水线架构

数据处理遵循单向流水线，职责清晰：

```
硬件采集 → 回绕差分 → 极性修正 → 物理量转换 → 滤波处理 → 快照输出
```

## 二、硬件认知：编码器模式下的TIM

### 2.1 核心原理

**编码器模式下，TIM 的时钟源是外部引脚信号，而非内部时钟树。**

|模式|时钟源|计数触发|ARR/PSC作用|
|---|---|---|---|
|普通定时器|内部时钟（PCLK）|时钟脉冲驱动CNT|决定定时周期|
|**编码器模式**|外部引脚（TI1/TI2）|A/B相边沿驱动CNT|ARR仅决定计数范围|

**结论**：

- 脉冲频率 = 编码器线数 × 转速(RPM) / 60 × 倍频数，**与时钟树无关**
- ARR 推荐设为 65535（最大计数范围）
- PSC 在编码器模式下无效
- 时钟树唯一作用：开启 TIM 外设时钟

### 2.2 倍频原理

TIM 硬件编码器模式可对 A/B 两相信号进行 1x/2x/4x 倍频：

```
1x 倍频：仅统计 A 相上升沿
2x 倍频：统计 A 相上升沿 + 下降沿
4x 倍频：统计 A/B 两相的所有边沿（推荐，精度最高）
```

### 3.2 关键阶段说明

#### Stage 2: 回绕差分

**问题**：CNT 从 65535 跳到 0 时，如何正确计算增量？

**原理**：利用补码特性，`int16_t` 强转自动处理溢出。

```
CNT: 65534 → 65535 → 0 → 1
uint16_t 差分: 0 - 65535 = 1 (错误，实际应为+1)
int16_t 强转后: 自动解释为 +1 (正确)
```

#### Stage 4b: Q16 定点数

**为什么不用 float？**

- Cortex-M4 虽有 FPU，但定点数更快
- 中断上下文更安全
- 无精度损失

**计算方法**：

```
每脉冲位移 = 轮周长 / 每转脉冲数
Q16值 = 每脉冲位移 × 65536

运行时：位移(mm) = (delta × Q16值) >> 16
```

#### Stage 5: IIR 滤波

**问题**：低速时 delta_count 在 0/1/2 间跳动，速度剧烈抖动。

**解决**：一阶惯性滤波 

**调参**：α 越大响应越快但噪声越大，建议从 0.3 起调。

## 四、关键概念

### 4.1 减速比

**定义**：减速比 = 电机转速 / 输出轴转速

**两大作用**：

|作用|公式|工程意义|
|---|---|---|
|扭矩放大|减速比|用速度换力量|
|精度提升|脉冲数线数倍频减速比|减速比越大，测量精度越高|

**选型原则**：让电机在目标最高速度时工作在额定转速（约60%空载转速）附近。

### 4.2 速度单位

|单位|适用场景|
|---|---|
|**mm/s**|小车运动控制，PID目标直观|
|**RPM**|电机规格书，电机驱动器|
|**rad/s**|动力学计算，IMU融合|

**换算核心**：`RPM = (delta × 60000) / (counts_per_rev × tick_ms)`

## 五、工程避坑指南

### 5.1 数据撕裂问题

**现象**：主循环读取结构体时被中断打断，读到"一半新一半旧"的数据。

**解决**：快照机制，关中断原子拷贝。

```c
void Encoder_TakeSnapshot(encoder_wheel_sample_t *snapshot) {
    __disable_irq();
    *snapshot = sample;  // 结构体拷贝
    __enable_irq();
}
```

### 5.2 乘法溢出

**风险**：`delta_count × mm_per_count_q16` 可能溢出 int32。

**解决**：使用 `int64_t` 中间变量。

### 5.3 异常检测

|异常类型|检测方法|处理策略|
|---|---|---|
|脉冲跳变|`|delta|
|堵转|`PWM满 && delta==0` 持续N次|报警停机|
|断线|长期无脉冲变化|报警|

### 5.4 滤波与响应的平衡

|采样周期|滤波系数α|效果|
|---|---|---|
|短 (5ms)|小 (0.1~0.2)|噪声大，需强滤波|
|长 (20ms)|大 (0.3~0.5)|自带平滑，滤波可弱|

**原则**：采样周期缩短时，α 应相应减小。

## 六、调试建议

1. **区分 raw 与 filt**：排查 PID 震荡时，同时打印原始速度和滤波速度，判断是测量噪声还是控制问题。
    
2. **验证极性**：手动转动轮子，观察 `pos_count_4` 变化方向是否符合预期。
    
3. **检查溢出**：高速运行时监控 `delta_count_4` 是否接近 `int16_t` 极限。
    
4. **单位一致性**：确保 PID 目标值与反馈值单位一致（同为 mm/s 或 RPM）。
