### 理解



1. 每个功能有独立执行节奏
2. 阻塞只影响当前线程
3. 高优先级控制任务可以抢占低优先级任务
4. 后期容易扩展消息队列、事件、互斥量


5. 采样线程只负责发送数据，不关心谁处理
6. 控制线程可以阻塞等待数据，不需要 while 里轮询
7. 队列自带缓冲，短时间处理慢一点不会立刻丢数据
8. 消息是拷贝传递，小结构体不用共享内存，暂时不需要 mutex
9. 后面扩展报警/日志时，模块边界更清楚

10. Shell 命令带参数怎么写
11. 字符串参数怎么转整数
12. 运行时修改系统状态
13. 控制线程和 shell 线程如何共享参数
14. 为什么目标参数也要用 mutex 保护

rt_mq    = 传递数据
rt_mutex = 保护共享状态
rt_event = 通知状态变化

### API

rt_thread_create(name, entry, parameter, stack_size, priority, tick);

name        线程名，RT_NAME_MAX=8，所以别太长
entry       线程入口函数
parameter   传给线程的参数，暂时不用就传 RT_NULL
stack_size  线程栈大小
priority    优先级，RT-Thread 里数字越小优先级越高
tick        同优先级线程的时间片
