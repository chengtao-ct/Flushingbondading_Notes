lcd_init()
  -> 配 GPIO/FSMC
  -> HAL_SRAM_Init()
  -> 读 LCD ID
  -> 根据 ID 调 lcd_ex_xxx_reginit()
  -> 设置方向
  -> 开背光
  -> 清屏

最重要的是这条调用链：

```
串口输入 igw_lcd_test
        ↓
调用 lcd_init()
        ↓
读取 LCD ID
        ↓
识别到 7789，也就是 ST7789
        ↓
调用 ST7789 初始化函数
        ↓
清屏、画色块、显示字符
```

也就是说，`LCD ID:7789` 不是随便打印的数字，它说明：

- MCU 和 LCD 的硬件总线基本通了。
- LCD 读 ID 功能正常。
- 驱动识别到了屏幕控制器。
- ST7789 初始化流程被执行。
- 后面的画图函数可以往屏幕显存写数据。


LCD 坐标一般这样理解：

```
左上角是 (0, 0)

x 向右增加
y 向下增加
```

所以：

```
lcd_fill(10, 110, 110, 160, RED);
```

意思是：

```
从左上角 (10, 110)
填充到右下角 (110, 160)
颜色是红色
```

    = sample_count > 0，snapshot 是真实更新后的状态

    = HMI 线程活着，但 sensor/control/model 链路没把 sample 写进模型

Seq 开始增长
    = snapshot 已经拿到真实包



还有一个重要实现细节：当前 `sensor_service` 把 SHT31 当成“整包门禁”，只要温湿度失败，就不发送任何 sample。对于你的工程目标，这个策略太脆弱。更合理的是：真实可用的数据都进入 sample；SHT31 失败只影响温湿度有效性，不应该阻断光照、雨水和 HMI 状态更新。


对，**SHT31 按道理读成功**和**HMI 仍然 `SEQ=0`**并不矛盾。因为现在真正要确认的是：

```
igw_sht31 命令能读成功
≠
sensor_service 线程正在周期读成功并发送 sample
≠
control_service 收到 sample 并写入 igw_model
```

也就是说，`igw_sht31` 只是证明这个函数在 shell 命令里能工作；但 HMI 的数据来源不是 `igw_sht31`，而是：

```
sensor_service -> rt_mq_send -> control_service -> igw_model_update_control -> hmi_service
```

现在屏幕 `SEQ=0` 只说明：

```
igw_model_update_control() 还没有成功执行过
```

可能问题点有 4 个：

SHT31 / 光照 / 雨水硬件
        ↓
board_port 读取底层数据
        ↓
sensor_service 组装 igw_sample_t
        ↓
sample_mq 消息队列
        ↓
control_service 接收 sample，计算 PWM
        ↓
igw_model_update_control() 更新模型快照
        ↓
hmi_service / igw_status 读取 snapshot



光照底层已经改成可校准了。接下来我加一个 shell 命令，让你不用每次改代码也能校准：`igw_light_cal show` 看当前亮/暗标定，`igw_light_cal dark` 在遮住时记录暗端，`igw_light_cal bright` 在强光下记录亮端。



是的，**第二阶段的核心数据闭环已经打通了**。

现在这条链路已经成立：

```
SHT31 / 光照 / 雨水
    ↓
board_port
    ↓
sensor_service
    ↓
sample_mq
    ↓
control_service
    ↓

    ↓

```

只要 HMI 能实时更新真实温湿度、光照、雨水、PWM、Seq，而且 `igw_status` 和屏幕一致，就说明“真实数据进入模型，再被界面显示”这一关已经闭环。

但如果按我们最初的“第二阶段：自制轻量触摸 HMI”完整标准来看，还剩几个收尾项：

1. **触摸按钮是否完成验证**  
    `MODE / T- / T+ / PWM- / PWM+` 是否能真实改变模型状态。
    
2. **串口状态与屏幕状态是否同步**  
    点击按钮后，`igw_status` 和 LCD 显示要一致。
    
3. **临时调试日志要收敛**  
    比如 `[sensor] mq=...`、`[control] recv ok...` 这些可以后续删掉或用 debug 宏包起来。
    
4. **光照校准可以继续优化**  
    现在已经能工作，但最好记录几组 `raw`，再确定最终 `bright/dark` 标定值。
