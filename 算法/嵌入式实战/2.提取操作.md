在单片机驱动 TFT LCD 屏幕时，为了省内存，颜色通常不是 24 位的，而是 **16 位 (RGB565)** 的： `红色(Red)` 占高 5 位，`绿色(Green)` 占中间 6 位，`蓝色(Blue)` 占低 5 位。 例如一个 16 位颜色值 `color = 0xF800`（纯红色）。

- **任务：** 给你一个 16 位的颜色变量 `uint16_t color`，请你写出三个表达式，利用**掩码 (Mask)** 和**右移 (`>>`)**，把里面的 `R`、`G`、`B` 三个通道的值分别剥离出来，存到三个 8 位变量 `uint8_t r, g, b` 中。
    
- **挑战点：** 比如你想拿中间的 `G`，你必须先造一个“只露中间 6 个洞”的面具（Mask）把它套出来，然后再把它平移到最右边对齐。


**思考**
- 关于掩码的话，肯定先设置掩码的操作,这自然让我想起了如何清0的操作，既然有清0的操作同时，也应该有提取的操作的define
- 提取肯定使用&，&上1是本身，&上0一定是0


```c
#define MASK(width,offest)     ~((1 << (width))-1 <<(offest))

// 提取宏定义：从 reg 中，提取从第 offset 位开始、宽度为 width 的数据
#define GET_BITS(reg, width, offset)   (((reg) >> (offset)) & ((1 << (width)) - 1))
```


```c
uint16_t color = 0xF800; // 纯红色测试 (二进制: 11111 000000 00000)

// 1. 提取红色 (最高 5 位)
// 动作：右移 11 位到底，然后套上 5位 的面具 (0x1F) 砍掉可能的垃圾高位
uint8_t r = (color >> 11) & 0x1F; 

// 2. 提取绿色 (中间 6 位)
// 动作：右移 5 位到底，然后套上 6位 的面具 (0x3F) 砍掉跟着移下来的红色
uint8_t g = (color >> 5)  & 0x3F;

// 3. 提取蓝色 (最低 5 位)
// 动作：本来就在最右边，不需要移位！直接套上 5位 的面具 (0x1F) 砍掉左边所有的红和绿
uint8_t b = color & 0x1F;
```


**第一步：`(reg) >> (offset)` —— 【平移出列】** 不管目标数据藏在多高的位置，直接往右平移 `offset` 位，把它硬生生拽到最右边（最低位）对齐。 _(此时，左边多余的垃圾数据也跟着移下来了，需要清理。)_

**第二步：`(1 << (width)) - 1` —— 【铸造砍刀】** 这个你太熟悉了！这就是上一题造面具的“第二层”逻辑。 比如宽度 `width` 是 6。 `1 << 6` 是 `1000000`（十进制的 64）。 减 1 后变成 `0111111`（十进制的 63，十六进制的 `0x3F`）。 我们成功造出了一把**“刚好有 6 个 1 的砍刀”**！

**第三步：`&` —— 【手起刀落】** 把平移下来的数据，和这把刚好够宽的砍刀做按位与（`&`）。 目标数据所在的位全都是 1，完美保留；左边跟着移下来的垃圾数据全遇到了 0，瞬间灰飞烟灭。
