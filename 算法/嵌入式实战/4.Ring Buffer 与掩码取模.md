这个技巧在串口 DMA 接收、操作系统任务调度、甚至是高频的电机控制中断（ISR）中被奉为神明。

**实战场景：** 你正在写一个串口的高速接收环形缓冲区 (Ring Buffer)。缓冲区的总大小被设定为 256 字节（`#define BUFFER_SIZE 256`）。 你有一个写入指针 `uint16_t write_idx = 0;`。 每次串口来一个字节，指针就加 1。当指针到达 256 时，它必须瞬间绕回到 0，实现“环形”覆盖。

利用位运算（`&`），写出一种**没有 `if` 分支、没有 `%` 取模**的极速绕回写法。 （**提示：** 为什么操作系统和底层库里的缓冲区大小，永远要求是 $2^N$？比如 64, 128, 256, 1024？思考一下，如果对 256 这种特殊的数字减去 1，得到的二进制会有什么极其美妙的特性？）



**思考**：
- 这不就是Leetcode里面的2的幂吗
- 不对不是，是leetcode里面的位1的个数


```c
int hammingWeight(uint32_t n) {
    int count = 0;
    while (n != 0) {
        n = n & (n - 1); // 极其冷酷：干掉最低位的 1
        count++;         // 人头数 + 1
    }
    return count;
}
```
