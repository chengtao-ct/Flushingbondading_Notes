> [!abstract] 题目概述
> **考察方向：** 同步递归 (Simultaneous Recursion)。
>
> **嵌入式场景：** 在工业冗余系统中，两块 A/B 备份板会各自生成一套状态树。你需要判定这两套系统是否互为"镜像对称"，以确保主备切换时的状态完全同步。
>
> **挑战点：** 这不是简单的"自己跟自己比"，而是要同时对比"左子树的左"和"右子树的右"。如何写出一个接收两个指针的递归函数？

> [!note] 思考过程
> **问题：**
> - 在最底层要处理的是什么信息，怎么才能使完全对称呢
> - 我的一开始的思路是错的，因为怎么对比都是内部对比
> - 一定需要截至条件吗？
>
> **思考：**
> - 写递归第一步先明确截至条件在哪里
> - 第二步思考怎么分发任务
> - 第三步思考每次分发任务让他干什么，层级处理信息
> - 每一步处理的信息，应该是左右返回的是什么东西，然后返回单一个，向上层汇报

## 解法

```c
bool isMirror(struct TreeNode* LeftNode , struct TreeNode* RightNode)
{
    // 第一关：必须先查"双空"。两人同时踩空，说明形状完美对称。
    if(LeftNode == NULL && RightNode == NULL) {
        return true;
    }

    // 第二关：再查"单空"。
    // 能漏到这一关，说明【绝对不可能两个都是空】。
    // 此时只要有任何一个是空，那必定是"一空一不空"，绝对不对称！
    if(LeftNode == NULL || RightNode == NULL) {
        return false;
    }

    // 第三关：两人都在，对暗号（值）
    if(LeftNode->val != RightNode->val) {
        return false;
    }

    // 第四关：暗号正确，X 型交叉向下继续查账
    return isMirror(LeftNode->left, RightNode->right) && 
           isMirror(LeftNode->right, RightNode->left);
}

bool isSymmetric(struct TreeNode* root) {
    if (root == NULL) {
        return true;
    }
    return isMirror(root->left, root->right);
}
```

> [!tip] 核心收获
> ### 破局心法：越苛刻的条件，越要放在前面！
> 在写 `if` 拦截器的时候，必须**先抓"双空"（最严苛），再抓"单空"（较宽容）**。
