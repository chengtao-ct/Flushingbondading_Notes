> [!abstract] 题目概述
> **考察方向：** 迭代法（非递归）遍历树结构、状态机思维。
>
> **嵌入式场景：** 昨天我们用递归（函数甩锅）算出了树的深度和对称性。但在资源极度匮乏的单片机（如只有 4KB RAM的 8051 或低端 STM8）中，递归调用极易导致**系统栈溢出（Stack Overflow）**宕机。老兵必须掌握不用递归，纯靠 `while` 循环和一个我们自己捏的数组栈（Stack），来手动模拟这棵树的遍历过程！
>
> **挑战点：** 如何用一个 `while` 循环，控制指针沿着树干一路向左下探，撞到南墙（NULL）后再回头打印？（极其锻炼空间想象力）。

## 解法

### 递归解法

```c
#include <stdlib.h>

// 包工头：辅助递归函数 (不返回任何东西，纯干活)
// 参数：当前节点、大老板给的数组、大老板给的游标指针
void inorderHelper(struct TreeNode* root, int* res, int* returnSize) {
    // 撞到南墙，原路返回
    if (root == NULL) {
        return;
    }

    // 1. 先疯狂向【左】下探
    inorderHelper(root->left, res, returnSize);

    // 2. 处理【根】节点：把值塞进数组，并且游标加 1
    res[*returnSize] = root->val;
    (*returnSize)++;

    // 3. 最后向【右】下探
    inorderHelper(root->right, res, returnSize);
}


// 大老板：LeetCode 给的入口函数
int* inorderTraversal(struct TreeNode* root, int* returnSize) {
    // 1. 出钱买地：题目说最多 100 个节点，我们直接申请 100 个 int 的空间
    int* res = (int*)malloc(sizeof(int) * 100);
    
    // 2. 初始化游标：一开始数组里有 0 个数字
    *returnSize = 0;
    
    // 3. 呼叫包工头干活！把树根、空数组、游标地址交给他
    inorderHelper(root, res, returnSize);
    
    // 4. 干完活了，包工头已经把 res 填满了，并且 *returnSize 也变成实际的节点数了
    // 大老板直接把数组上交！
    return res;
}
```

### 迭代解法

```c
int* inorderTraversal(struct TreeNode* root, int* returnSize) {
    // 1. 建大本营
    int* res = (int*)malloc(sizeof(int) * 100);
    *returnSize = 0;

    // 2. 建探险家的记事本 (模拟栈)
    struct TreeNode** stack = (struct TreeNode**)malloc(sizeof(struct TreeNode*) * 100);
    int top = 0;

    struct TreeNode* current = root;

    // 3. 开始探险
    while(current != NULL || top > 0) {
        
        if(current != NULL) {
            // 有路：疯狂向左，沿途记账
            stack[top] = current;
            top++;
            current = current->left;
        } else {
            // 撞墙回头：出栈，记录，并向右走
            top--;
            current = stack[top];
            res[*returnSize] = current->val;
            (*returnSize)++;
            current = current->right;
        }
    }
    
    // 4. 打扫战场并上交成果！
    free(stack); 
    return res;
}
```

> [!tip] 核心收获
> **递归 vs 迭代的本质区别：** 一个是硬件栈（CPU 自动管理的调用栈），一个是软件栈（你自己用数组模拟的栈）。迭代法就是你亲手扮演 SP 指针，手动管理压栈和出栈的过程。两者做的事情完全一样，但迭代法让你对内存有绝对的控制权——在嵌入式里，这就是生与死的区别。
