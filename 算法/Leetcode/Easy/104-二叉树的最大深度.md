> [!abstract] 题目概述
> **考察方向：** 深度优先搜索（DFS）与递归（Recursion）的极致精简。
>
> **嵌入式场景：** 你正在用 ESP32 开发一个带屏幕的智能家居中控面板。菜单是分级的（主菜单 -> 灯光控制 -> 卧室灯 -> 亮度调节）。你需要写一个算法来评估这个 UI 菜单树的"最大层级"，以确保用户的操作路径在安全范围内，并防止层级太深导致微控制器的栈内存（Stack）溢出。
>
> **挑战点：** 抛弃你之前写 `for` 和 `while` 的习惯。如何用不超过 5 行的纯递归代码，优雅地算出左右子树的深度？

> [!note] 思考过程
> **本质：** 自顶向下甩锅，自底向上汇报
>
> **思考：**
> - 因为你用while的化有一个非常明显的问题，就是如何判断他是最大深度
> - 对于递归就是函数的重复调用嘛？那么你现在要思考如何设计这个函数，栈的进出
> - 对于递归我感觉你一定要画图，不画图，非常难以思考
>
> **递归本质：**
> - 自顶向下甩锅，自底向上汇报
> - 所以说一定要分层思考，不要停在单一层面
> - 去思考终止的逻辑，去思考递归的元素，去思考每一层递归是为了添加什么，返回什么
>
> **问题：**
> - 递归是如何实现的
> - 没什么思路

## 解法

```c
int maxDepth(struct TreeNode* root) {
    // 1. 甩锅终点：如果是空节点，深度为 0
    if (root == NULL) {
        return 0;
    }

    // 2. 疯狂甩锅：问左手下和右手下
    int left_depth = maxDepth(root->left);
    int right_depth = maxDepth(root->right);

    // 3. 汇总汇报：挑个大的，加上自己这 1 层，往上报
    return (left_depth > right_depth ? left_depth : right_depth) + 1;
}
```

> [!tip] 核心收获
> 递归的本质就是"甩锅与汇报"——自顶向下把任务分发到最底层，自底向上把结果逐层汇总。写递归三步走：先想终止条件，再想怎么分发任务，最后想每一层返回什么。
