> [!abstract] 题目概述
> **考察方向：** 极致的位运算优化。
> **嵌入式场景：** 内存对齐（Alignment）检查。很多硬件 DMA 传输要求数据地址必须是 2 的整数次幂。
> **挑战点：** 昨天我们用 `for` 循环移位翻转了寄存器。这道题，你能否不用任何循环，**只用 1 行代码** 就判断出一个无符号整数是不是 2 的幂？（提示：想想 2 的幂在二进制下长什么样，比如 2、4、8、16）。

> [!note] 思考过程
> - **想法**：死都想不明白，不遍历怎么操作。

> [!note] Kernighan 算法原理
> 我们在第一天（D1）提到过一个内核级优化技巧：**Kernighan 算法**。它的核心公式是 `n & (n - 1)`。
>
> 你仔细观察一下 2 的幂在二进制下的特征：
> - $2$ = `0000 0010`
> - $4$ = `0000 0100`
> - $8$ = `0000 1000`
>
> **发现了吗？凡是 2 的幂，它的二进制位中永远只有 1 个 `1`，其余全为 `0`。**
>
> 现在，如果我们把一个 2 的幂减去 1，会发生什么？
> - $8$ = `0000 1000`
> - $7$ = `0000 0111` （也就是 8 - 1）
>
> 如果你把它们做**按位与（`&`）**运算：
> `8 & 7` -> `1000 & 0111` = `0000 0000` (也就是 0)！

## 解法

### 暴力解法（循环除以2）

```c
bool isPowerOfTwo(int n) {
    if(n==1) return 1;
    if(n<=0) return 0;
    while(n%2==0){
        n=n/2;
    }
    return (n==1);
    
}
```

### 位运算一行秒杀

```c
    return (n > 0) && ((n & (n - 1)) == 0);
```

> [!tip] 核心收获
> 从这两个答案就能看出来，你到底是站在人的角度思考，还是站在汇编的角度思考问题。
