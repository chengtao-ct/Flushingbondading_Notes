> [!abstract] 题目概述
> **考察方向：** 极致的位运算（Bit Manipulation）压榨。
> **嵌入式场景：** 你读取了一个 32 位的硬件状态寄存器（Register），每一位（Bit）代表一个外设传感器的报警状态（1表示报警，0表示正常）。你需要写一个以最快速度统计"当前共有几个传感器报警"的函数。
> **挑战点：** 你当然可以用 `for` 循环移位 32 次慢慢数，但这太慢了！还记得我们在 Day 2 讲过的那个**"1 行代码秒杀 2 的幂"**的绝招吗？（提示：`n & (n - 1)`）。今天我们要用它来"降维打击"这道题。

> [!note] 思考过程
> - 其实我的第一想法也是用for循环，用移位符号和&符号解决这个问题，我看了一下他的答案，好吧我也想不到，`n&(n-1)` 会自动消除一个1，nb，Brian Kernighan 算法（位运算降维打击）。
> - 其实我写了几天的算法以后，我有一个清晰的认识，先列数字，尝试例子，多画图，一开始很难有思路的，一些关于堆啊、栈啊、指针要有基础的认知。

## 解法

```c
int hammingWeight(uint32_t n) {
    int count = 0;          // 准备一个计数器
    
    while (n != 0) {        // 只要 n 还没被榨干成 0
        n = n & (n - 1);    // 扣动扳机：抹掉最右边的一个 1
        count++;            // 击杀数 + 1
    }
    
    return count;           // 完工，汇报击杀数！
}
```

> [!tip] 核心收获
> Brian Kernighan 算法：`n & (n - 1)` 每次消除最右边的一个 1，循环次数等于 1 的个数，远优于遍历全部 32 位。
