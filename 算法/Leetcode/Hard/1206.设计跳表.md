> [!abstract] 题目概述
> **考察方向：** 跳表（Skiplist）的查找、插入、删除，以及多层链表指针的维护。
>
> **嵌入式场景：** 跳表可以看成“有序链表 + 多级索引”。如果一个系统里有大量按时间排序的对象，例如软件定时器、超时任务、延迟队列，普通有序链表每次查找插入都可能退化成 $O(n)$；跳表用随机层级换取期望 $O(\log n)$，思路和“给长链表修几层快速通道”非常像。
>
> **挑战点：** 这道题不是难在语法，而是难在三个心智模型：`head` 为什么拥有所有层、`randomLevel()` 什么时候决定节点高度、`update` 数组到底记录了什么。

> [!note] 原来的卡点
> - 高架桥的高度是插入时重新决定，还是一开始全部定好？
> - 跳表为什么天然有序？是不是需要先排序？
> - 每层的起始节点从哪里来？
> - 查找时“往右走、往下掉”的逻辑为什么成立？
> - 删除节点时，是每一层都要 `free`，还是只释放一次？
>
> **现在的答案：**
> - `head` 在初始化时就拥有 `MAX_LEVEL` 个 `forward` 指针，作为所有楼层的统一起点。
> - 每个真实节点在 `add` 的那一刻，通过 `randomLevel()` 决定自己的高度；老节点高度不会再变化。
> - 有序性不是事后排序出来的，而是每次插入时都找到正确前驱，把新节点插到有序位置上。
> - 查找逻辑的本质是不断缩小区间：能往右逼近就往右，右边过头或为空就降一层。
> - 一个节点在内存里只 `malloc` 一次，所以删除时只需要顺着 Level 0 找到它，断开所有层的指针后 `free` 一次。

## 核心模型

一句话：**跳表 = 有序单链表 + 多层快速索引。**

最底层 `Level 0` 保存所有真实节点；越高层节点越少，只保留一部分“站点”作为快速通道。

```text
Level 3: HEAD --------------------------> 90
Level 2: HEAD ------------> 40 --------> 90
Level 1: HEAD ----> 10 ----> 40 -> 60 -> 90
Level 0: HEAD -> 10 -> 20 -> 40 -> 45 -> 60 -> 90
```

注意这里不是“每层重新建一条完全独立的链表”。一个节点如果高度是 3，就说明它的 `forward[0]`、`forward[1]`、`forward[2]` 都可能参与连接；它在内存里仍然只是一个节点。

```c
typedef struct skiplist_node {
    int val;
    struct skiplist_node *forward[MAX_LEVEL];
} SkiplistNode;
```

这里的 `forward[i]` 就是“这个节点在第 i 层的下一个节点”。`forward[0]` 是地面链表，必须完整串起所有节点。

## 为什么它是有序的

跳表不会先随便插入，再整体排序。它和有序链表一样，**插入前先找位置**。

对于要插入的 `num`，我们在每一层都只往右走到“最后一个小于 `num` 的节点”。这个节点就是该层的前驱。最后把新节点接到这些前驱后面，整个结构就继续保持有序。

所以有序性来自这个不变量：

> 每一层从左到右都是递增的；`Level 0` 包含全部元素，高层只是 `Level 0` 的稀疏索引。

这也是它适合定时器、延迟队列、排行榜、Redis 有序集合这类场景的原因：业务本身就有一个排序键。

## 统一寻址逻辑

`search`、`add`、`erase` 的第一步其实都是同一个动作：从最高层开始找前驱。

口诀只有两句：

1. 右边节点存在，并且右边的值 `< target`，就继续往右走。
2. 右边为空，或者右边的值 `>= target`，说明本层不能再走了，记录当前位置，然后往下一层。

代码骨架如下：

```c
SkiplistNode *curr = obj->head;

for (int level = MAX_LEVEL - 1; level >= 0; level--) {
    while (curr->forward[level] != NULL &&
           curr->forward[level]->val < target) {
        curr = curr->forward[level];
    }

    update[level] = curr; // add/erase 需要；search 可以不记录
}
```

这里最关键的是 `update` 数组：

> `update[level]` 表示在第 `level` 层，目标值前面的最后一个节点。

普通链表插入和删除，最重要的是找到“前驱节点”。跳表只是把这个问题放大了：因为它有很多层，所以你要同时记住每一层的前驱节点。

## 三大操作

### 1. 查找 search

查找只需要“找前驱”，最后检查地面层的下一个节点。

```text
从最高层开始：
  能右移就右移
  不能右移就下降
下降到 Level 0 后：
  curr->forward[0] 就是第一个 >= target 的节点
```

如果 `curr->forward[0] != NULL && curr->forward[0]->val == target`，说明找到了。

### 2. 插入 add

插入分三步：

1. 用统一寻址逻辑填好 `update[MAX_LEVEL]`。
2. 用 `randomLevel()` 决定新节点高度。
3. 在 `0 ~ newLevel - 1` 层执行普通单链表插入。

单层插入的顺序要背熟：

```c
newNode->forward[level] = update[level]->forward[level];
update[level]->forward[level] = newNode;
```

先让新节点接住后面的链，再让前驱指向新节点。这个顺序和普通链表完全一样。

### 3. 删除 erase

删除也分三步：

1. 用统一寻址逻辑填好 `update[MAX_LEVEL]`。
2. 检查 `update[0]->forward[0]` 是不是要删除的值。
3. 如果是，就从低层往高层逐层断开它，最后 `free(targetNode)` 一次。

为什么只 `free` 一次？因为所谓“多层节点”不是多个节点，而是一个结构体里有多个 `forward` 指针。你断开的只是不同楼层的索引线，内存实体只有一块。

## randomLevel 的本质

`randomLevel()` 是“抛硬币盖楼”：

- 所有节点 100% 拥有 `Level 0`。
- 每多盖一层，都要再赢一次 50% 概率。
- 直到抛失败，或者达到 `MAX_LEVEL`。

因此：

- 大部分节点只有 1 层。
- 少部分节点有 2 层。
- 极少数节点能冲到很高层。

这就自然形成了“底层密、高层稀”的快速通道。

## C 语言实现

```c
#include <stdbool.h>
#include <stdlib.h>

#define MAX_LEVEL 16

typedef struct skiplist_node {
    int val;
    struct skiplist_node *forward[MAX_LEVEL];
} SkiplistNode;

typedef struct {
    SkiplistNode *head;
} Skiplist;

static SkiplistNode *createNode(int val) {
    SkiplistNode *node = (SkiplistNode *)malloc(sizeof(SkiplistNode));
    node->val = val;

    for (int i = 0; i < MAX_LEVEL; i++) {
        node->forward[i] = NULL;
    }

    return node;
}

static int randomLevel(void) {
    int level = 1;

    while ((rand() % 2 == 1) && level < MAX_LEVEL) {
        level++;
    }

    return level;
}

Skiplist *skiplistCreate(void) {
    Skiplist *obj = (Skiplist *)malloc(sizeof(Skiplist));
    obj->head = createNode(-1);
    return obj;
}

bool skiplistSearch(Skiplist *obj, int target) {
    SkiplistNode *curr = obj->head;

    for (int level = MAX_LEVEL - 1; level >= 0; level--) {
        while (curr->forward[level] != NULL &&
               curr->forward[level]->val < target) {
            curr = curr->forward[level];
        }
    }

    curr = curr->forward[0];
    return curr != NULL && curr->val == target;
}

void skiplistAdd(Skiplist *obj, int num) {
    SkiplistNode *update[MAX_LEVEL];
    SkiplistNode *curr = obj->head;

    for (int level = MAX_LEVEL - 1; level >= 0; level--) {
        while (curr->forward[level] != NULL &&
               curr->forward[level]->val < num) {
            curr = curr->forward[level];
        }

        update[level] = curr;
    }

    int newLevel = randomLevel();
    SkiplistNode *newNode = createNode(num);

    for (int level = 0; level < newLevel; level++) {
        newNode->forward[level] = update[level]->forward[level];
        update[level]->forward[level] = newNode;
    }
}

bool skiplistErase(Skiplist *obj, int num) {
    SkiplistNode *update[MAX_LEVEL];
    SkiplistNode *curr = obj->head;

    for (int level = MAX_LEVEL - 1; level >= 0; level--) {
        while (curr->forward[level] != NULL &&
               curr->forward[level]->val < num) {
            curr = curr->forward[level];
        }

        update[level] = curr;
    }

    SkiplistNode *targetNode = update[0]->forward[0];
    if (targetNode == NULL || targetNode->val != num) {
        return false;
    }

    for (int level = 0; level < MAX_LEVEL; level++) {
        if (update[level]->forward[level] != targetNode) {
            break;
        }

        update[level]->forward[level] = targetNode->forward[level];
    }

    free(targetNode);
    return true;
}

void skiplistFree(Skiplist *obj) {
    SkiplistNode *curr = obj->head;

    while (curr != NULL) {
        SkiplistNode *next = curr->forward[0];
        free(curr);
        curr = next;
    }

    free(obj);
}

/**
 * Your Skiplist struct will be instantiated and called as such:
 * Skiplist* obj = skiplistCreate();
 * bool param_1 = skiplistSearch(obj, target);
 * skiplistAdd(obj, num);
 * bool param_3 = skiplistErase(obj, num);
 * skiplistFree(obj);
 */
```


`forward[i]` 不是指向"数组"，而是指向"另一个节点结构体"。那个结构体自己也有 `forward[]`，如此递归下去，就在每一层形成了链表结构。节点在内存中只有一份，"层"只是指针串联出来的逻辑视图。

## 易错点

> [!warning] 必须盯住的 C 语言细节
> 1. `for (int level = MAX_LEVEL - 1; level >= 0; level--)` 不能写成 `level > 0`，否则会漏掉 `Level 0`。
> 2. `malloc(sizeof(SkiplistNode))`，不能写成 `sizeof(SkiplistNode *)`。
> 3. 新节点的 `forward[]` 必须初始化为 `NULL`。
> 4. `search/add/erase` 的横向移动条件都是 `< target` 或 `< num`，不是 `<=`。这样才能停在目标值的前驱节点。
> 5. LeetCode 允许重复插入。删除时只删除找到的第一个匹配节点。
> 6. 释放内存时沿着 `forward[0]` 走就够了，因为所有真实节点都一定在 `Level 0`。

## 嵌入式视角

> [!tip] 工程判断
> - 如果节点数量很小，普通有序链表更简单、更确定，没必要上跳表。
> - 如果节点很多，并且频繁按 key 插入、删除、查找，跳表比普通链表更适合。
> - 在 MCU/RTOS 里不要随便在中断里 `malloc/free`。真要用跳表，优先考虑内存池。
> - 当前 LeetCode 写法为了简单，每个节点都固定带 `MAX_LEVEL` 个指针，空间是 `O(n * MAX_LEVEL)`。工业实现可以用柔性数组、分级内存池，或者把节点高度也存下来，减少 RAM 浪费。
> - `rand()` 在刷题里够用，但在嵌入式工程里要考虑确定性、可复现性和随机源质量。

## 复杂度

- 平均查找：$O(\log n)$
- 平均插入：$O(\log n)$
- 平均删除：$O(\log n)$
- 最坏情况：$O(n)$，因为随机层级理论上可能退化
- 空间复杂度：刷题固定数组写法是 $O(n \cdot MAX\_LEVEL)$；按实际高度分配时，期望是 $O(n)$

## 复习清单

> [!example]- 背诵版
> 1. 跳表底层是完整有序链表，高层只是索引。
> 2. `head` 从初始化开始就拥有所有层，真实节点插入时才随机决定高度。
> 3. 查找口诀：能右移就右移，不能右移就下楼。
> 4. `update[i]` 是第 `i` 层目标位置的前驱节点。
> 5. 插入口诀：先让新节点接住后面，再让前驱指向新节点。
> 6. 删除口诀：先用 `update` 找到每层前驱，再逐层绕过目标节点。
> 7. 释放口诀：沿着 `Level 0` 一个个 `free`，每个节点只释放一次。
> 8. 嵌入式口诀：数据少用链表，数据多且频繁有序操作才考虑跳表；实时场景慎用动态内存。
