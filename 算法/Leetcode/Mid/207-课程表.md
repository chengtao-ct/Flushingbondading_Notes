- **嵌入式场景：** 假设你正在为一个复杂的工业主板写 Bootloader（启动代码）。板子上有 10 个外设需要初始化。但是，I2C 必须在 DMA 之后启动，DMA 必须在时钟（Clock）之后启动，而时钟又依赖电源管理芯片。你如何写一个算法，给出一个**绝对安全的启动顺序**？更致命的是，如果硬件工程师画错了图，导致 A 依赖 B，B 依赖 A（死锁环），你的算法如何第一时间报警？
    
- **挑战点：** 这是你第一次真正接触“图”结构。如何用数组把错综复杂的依赖关系表示出来（邻接表）？如何像剥洋葱一样，从“没有任何依赖”的节点一层层往里剥



**问题**：
- 第一个事情：图这个结构，是不是在Rt-thread的优先级调度里面存在的啊，我记得它是用数组+双向链表完成一些任务的
- 依赖关系没有一点思路啊
- 一开始对多维数组理解不够充分



**解法**
- Kahn 算法（拓扑排序）


要想用 C 语言实现这个过程，你需要准备三样东西：

1. **入度账本（数组）：** `int inDegree[numCourses] = {0};` 用来记录每个任务当前被几个条件卡着。
    
2. **小本本（邻接表）：** 因为当你执行完任务 C 时，你需要通知 B 和 D（它们的入度减 1）。所以你需要记录**“我执行完后，该去通知谁”**。
    
3. **就绪队列（数组模拟）：** `int queue[numCourses];` 把所有入度为 0 的任务扔进去，然后排队一个个执行。




#### 入度表 (In-Degree Array) —— “我的紧箍咒”

- **本质：** 它是一个**向内看**的数据结构。
    
- **它回答的问题：** “我现在能马上开始干活吗？”
    
- **怎么用：** 员工 B 每天早上来看一眼自己的入度表 `inDegree[B]`。如果上面的数字是 2，代表他头上还有两个领导（前置任务）没发话，他只能干等（不能进队列）。他**根本不关心**是哪两个领导卡着他，他只关心“数字什么时候变成 0”。只要数字是 0，他立刻冲进“就绪队列”开始干活！
    

#### 2. 邻接表 (Adjacency List) —— “我的通知名单”

- **本质：** 它是一个**向外看**的数据结构。
    
- **它回答的问题：** “我干完活了，我该去通知谁？”
    
- **怎么用：** 员工 A 终于把活干完了。他立刻翻开自己的小本本 `adj[A]`，发现上面写着 `[B, C]`。他瞬间明白：“我要去解开 B 和 C 头上的紧箍咒！”他**根本不关心** B 和 C 还有没有被别人卡着，他只负责去告诉他们：“我这关你过了”。




**思路**
- 其实他的大体思路是先看到这个，要思考出来用什么来解决，这里用的拓扑排序
- 然后但是这个题目的形成完全不是拓扑算法，所以说你要架构成拓扑算法
- 形成入度表和邻接表，
- 最后才是调度过程




```c
bool canFinish(int numCourses, int** prerequisites, int prerequisitesSize, int* prerequisitesColSize) {
    
    // ==========================================
    // 第一阶段：准备大锅与账本 (建表)
    // ==========================================
    int inDegree[2000] = {0};       // 紧箍咒表：记录每个人被几把锁卡着
    int adj[2000][2000] = {0};      // 通知名单：记录每个人干完活要去通知谁
    int adjCount[2000] = {0};       // 记录每个人的通知名单上有几个名字

    // ==========================================
    // 第二阶段：拉抽屉，分析依赖 (填表)
    // ==========================================
    for (int i = 0; i < prerequisitesSize; i++) {
        int target = prerequisites[i][0]; // 小弟 (被卡着的人)
        int pre    = prerequisites[i][1]; // 大佬 (卡别人的人)
        
        inDegree[target]++;                 // 小弟头上多了一把锁
        adj[pre][adjCount[pre]++] = target; // 大佬把小弟写进自己的通知名单
    }

    // ==========================================
    // 第三阶段：准备干活队列
    // ==========================================
    int queue[2000];
    int head = 0, tail = 0;

    // 🕵️‍♂️ 全公司大搜查：找到那些一开始就【没有任何锁】的自由人，直接踢进队列！
    for (int i = 0; i < numCourses; i++) {
        if (inDegree[i] == 0) {
            queue[tail++] = i; 
        }
    }

    // ==========================================
    // 第四阶段：流水线开始运转 (剥洋葱核心逻辑)
    // ==========================================
    int completedCount = 0; // 记录总共完成了几个任务

    // 只要队列里还有人，就一直干下去！
    while (head < tail) {
        // 1. 从队列开头拉出一个人来干活
        int current = queue[head++]; 
        completedCount++; // 业绩 + 1
        
        // 2. 他干完活了，翻开自己的《通知名单》，挨个去通知下游小弟！
        for (int i = 0; i < adjCount[current]; i++) {
            
            int nextCourse = adj[current][i]; // 找到名单上的一个小弟
            inDegree[nextCourse]--;           // 划掉这个小弟头上的一把锁
            
            // 3. 关键判定：如果小弟头上的锁全被划掉了（变成0了）
            if (inDegree[nextCourse] == 0) {
                queue[tail++] = nextCourse; // 立刻把小弟踢进队列，准备干活！
            }
        }
    }

    // ==========================================
    // 第五阶段：终极死锁判定
    // ==========================================
    // 如果没有死锁，每个人都会排队干完活，业绩 == 总人数。
    // 如果有死锁（A等B，B等A），他们的锁永远解不开，进不了队列，业绩就不达标！
    return completedCount == numCourses; 
}
```
