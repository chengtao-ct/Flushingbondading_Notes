> [!abstract] 题目概述
> **考察方向：** Floyd 判圈算法的进阶版（寻找环入口）。
>
> **嵌入式场景：** 昨天我们判定了内存池有没有"死环"。今天我们要升级任务：如果发现了死环，请精准定位出是**哪一个内存块**出了问题（环的起点）。
>
> **挑战点：** 这是一道带有数学证明色彩的题。当快慢指针相遇后，为什么从起点再派出一个指针，和慢指针一起跑，它们就一定会在环入口相遇？

> [!note] 思考过程
> 问题：在比较中不比较val，比较物理地址更加准确。
>
> 这个数学公式死都想不出来，顶多可以证明出来，想是想不到的。

> [!warning] 踩坑记录
> 比较的是物理地址（指针），不是节点的 val 值。

## 解法

![](assets/142-环形链表%20II/739e3b5c036ab32dbfbfabcfd003af0d.jpg)

```c
struct ListNode *detectCycle(struct ListNode *head) {
    if(head == NULL || head->next == NULL) {
        return NULL;
    }

    struct ListNode *p_slow = head;
    struct ListNode *p_fast = head;

    while(p_fast != NULL && p_fast->next != NULL) {
        p_slow = p_slow->next;
        p_fast = p_fast->next->next;

        if(p_slow == p_fast) {
            struct ListNode *p_index = head;

            while(p_index != p_slow) {
                p_index = p_index->next;
                p_slow = p_slow->next;
            }

            return p_index;
        }
    }

    return NULL;
}
```

> [!tip] 核心收获
> A=C 定理：龟兔相遇后，从起点再派出一个新兵和乌龟同速走，它们必然在环入口相遇。比较的是物理地址，不是 val。
