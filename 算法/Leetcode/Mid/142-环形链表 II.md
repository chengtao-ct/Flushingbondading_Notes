- **考察方向：** **Floyd 判圈算法的进阶版（寻找环入口）**。
    
- **嵌入式场景：** 昨天我们判定了内存池有没有“死环”。今天我们要升级任务：如果发现了死环，请精准定位出是**哪一个内存块**出了问题（环的起点）。
    
- **挑战点：** 这是一道带有数学证明色彩的题。当快慢指针相遇后，为什么从起点再派出一个指针，和慢指针一起跑，它们就一定会在环入口相遇？



**问题**：

- 在比较中不比较val
- 比较物理地址更加准确

**思考**：这个数学公式死都想不出来，顶多可以证明出来，想是想不到的

```c

struct ListNode *detectCycle(struct ListNode *head) {
    // 1. 防呆拦截
    if(head == NULL || head->next == NULL) {
        return NULL; // LeetCode 要求返回 NULL
    }

    struct ListNode *p_slow = head;
    struct ListNode *p_fast = head;

    // 2. 第一阶段：龟兔赛跑找相遇点
    while(p_fast != NULL && p_fast->next != NULL) {
        p_slow = p_slow->next;
        p_fast = p_fast->next->next;
 
        // 抓到了！有环，且在 p_slow 处相遇
        if(p_slow == p_fast) {
            
            // 3. 第二阶段：动用 A=C 定理找入口
            struct ListNode *p_index = head; // 派出一个新兵站在起点
            
            // 新兵和乌龟一起走，每次 1 步。比较物理地址！
            while(p_index != p_slow) {
                p_index = p_index->next;
                p_slow = p_slow->next;
            }
            
            // 当它们物理地址相同时，就是相遇在环入口了
            return p_index; // 返回这个入口节点的指针
        }
    }

    // 兔子摸到了 NULL，说明是直路，没有环
    return NULL;
}
```
