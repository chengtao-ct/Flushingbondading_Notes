> [!abstract] 题目概述
> **考察方向：** 哈希表（Hash Map）与 双向链表（Doubly Linked List）的史诗级缝合。
>
> **嵌入式场景：** 你正在用 ESP32 驱动一块漂亮的 LCD 彩屏。由于内部 RAM 极小，无法把所有 UI 图片都塞进内存，你只能用到哪张就去慢速的 SD 卡里读，然后缓存在 RAM 里。当 RAM 满了怎么办？**LRU（最近最少使用）算法**是工业界的标准解法：永远把最近刚刚看过的图片放在最前面，当容量爆满时，无情地踢掉最久没被访问过的那张图。
>
> **挑战点：** 题目极其变态地要求所有操作（读取、插入、踢人）都必须在 $O(1)$ 的时间复杂度内完成！

> [!note] 思考过程
> 问题：双向链表又是如何构建的？结构体的初始化，按逻辑来说，应该有一个数组一个链表吧？为什么哈希表的数组是以指针为基准的？不知道如何删除哈希表的节点？对于初始化的逻辑也要有点基础的认知？主要是感觉在KEY的这个逻辑上处理的有点问题？还有就是内存的释放不熟练。
>
> 思考：其实这个的本质架构就是RT-thread里面的优先级调度链表的实现逻辑。其实这个怎么说呢？就是我感觉在Mid的题目了，一般都是先进行造轮子，找到必要的数据结构和模块，再是去思考模块的融合。所以这个LRU缓存机制，一共有两个基本的数据结构，哈希表和双向链表。先完成最基本的链表的基础逻辑-操作，四步握手法的操作。主要是以前没有写过双向链表有点不熟悉。老是在链表里面用for循环来消除数据，我真是傻逼。

## 解法

```c
#include <stdlib.h>

typedef struct DLinkedNode {
    int key;
    int value;
    struct DLinkedNode* prev;
    struct DLinkedNode* next;
} DLinkedNode;

typedef struct {
    int capacity;
    int size;
    DLinkedNode* head;
    DLinkedNode* tail;
    DLinkedNode* hash[10001];
} LRUCache;

void addNode(DLinkedNode* head, DLinkedNode* node) {
    node->prev = head;
    node->next = head->next;
    head->next->prev = node;
    head->next = node;
}

void removeNode(DLinkedNode* node) {
    node->prev->next = node->next;
    node->next->prev = node->prev;
}

void moveToHead(DLinkedNode* head, DLinkedNode* node) {
    removeNode(node);
    addNode(head, node);
}

DLinkedNode* removeTail(DLinkedNode* tail) {
    DLinkedNode* evicted = tail->prev;
    removeNode(evicted);
    return evicted;
}

LRUCache* lRUCacheCreate(int capacity) {
    LRUCache* obj = (LRUCache*)malloc(sizeof(LRUCache));
    obj->capacity = capacity;
    obj->size = 0;

    obj->head = (DLinkedNode*)malloc(sizeof(DLinkedNode));
    obj->tail = (DLinkedNode*)malloc(sizeof(DLinkedNode));

    obj->head->prev = NULL;
    obj->head->next = obj->tail;
    obj->tail->prev = obj->head;
    obj->tail->next = NULL;

    for(int i = 0; i < 10001; i++) {
        obj->hash[i] = NULL;
    }
    return obj;
}

int lRUCacheGet(LRUCache* obj, int key) {
    DLinkedNode* node = obj->hash[key];

    if (node == NULL) {
        return -1;
    }

    moveToHead(obj->head, node);
    return node->value;
}

void lRUCachePut(LRUCache* obj, int key, int value) {
    DLinkedNode* node = obj->hash[key];

    if (node != NULL) {
        node->value = value;
        moveToHead(obj->head, node);
        return;
    }

    DLinkedNode* newNode = (DLinkedNode*)malloc(sizeof(DLinkedNode));
    newNode->key = key;
    newNode->value = value;

    obj->hash[key] = newNode;
    addNode(obj->head, newNode);
    obj->size++;

    if (obj->size > obj->capacity) {
        DLinkedNode* tailNode = removeTail(obj->tail);
        obj->hash[tailNode->key] = NULL;
        free(tailNode);
        obj->size--;
    }
}

void lRUCacheFree(LRUCache* obj) {
    DLinkedNode* curr = obj->head;
    while (curr != NULL) {
        DLinkedNode* temp = curr->next;
        free(curr);
        curr = temp;
    }
    free(obj);
}

/**
 * Your LRUCache struct will be instantiated and called as such:
 * LRUCache* obj = lRUCacheCreate(capacity);
 * int param_1 = lRUCacheGet(obj, key);
 * lRUCachePut(obj, key, value);
 * lRUCacheFree(obj);
 */
```

## 原理剖析

### 四步握手法（双向链表头插法）

#### 第一步：安顿新人的左手

新人 `B` 刚来，第一件事是赶紧抱住老大 `Head` 的大腿。

- **动作：** 让 `B` 的 `prev` 指针指向 `Head`。
- **C语言：** `node->prev = head;`

#### 第二步：安顿新人的右手

新人 `B` 还要拉住原来排第一的 `A`。注意，此时 `Head` 的右手依然还牵着 `A`，所以我们可以通过 `head->next` 找到 `A`。

- **动作：** 让 `B` 的 `next` 指针指向 `A` (`head->next`)。
- **C语言：** `node->next = head->next;`
- _(此时，新人 `B` 已经稳稳地抓住了两边，但旧的连线还没断。)_

#### 第三步：原来老二（A）的左手换人

`A` 发现新人 `B` 已经到位了，于是 `A` 松开抓着老大 `Head` 的左手，改去抓新人 `B`。

- **动作：** 找到 `A` (`head->next`)，把它的 `prev` 指针改指给 `node`。
- **C语言：** `head->next->prev = node;`
- **⚠️ 极度危险警告：** 这一步必须在第四步之前做！如果你先把 `head->next` 改成了别人，你就永远找不到 `A` 了！

#### 第四步：老大（Head）的右手换人

既然小弟们都已经牵好手了，老大 `Head` 终于可以松开原来抓着 `A` 的右手，改去牵新人 `B` 了。

- **动作：** 让 `Head` 的 `next` 指针指向 `node`。
- **C语言：** `head->next = node;`

**最终状态：** `[Head] <===> [B] <===> [A]`

老兵的终极秘籍是：**永远在链表两端放两个绝对不会被删除的"哨兵"：虚拟 Head 和虚拟 Tail。** 初始状态：`Head <==> Tail`

> 其实这道题的逻辑不复杂，只要一点一点的写，还是很好写的。本质架构就是RT-thread里面的优先级调度链表的实现逻辑。Mid题目的一般思路：先造轮子（必要的数据结构和模块），再思考模块的融合。
