模块 5：触发系统（Trigger）——示波器的"灵魂"
---
5.1 为什么触发是示波器最重要的概念？
同学，到现在为止你已经学了电压参数和时间参数，也亲手操作了示波器的基本设置。但你可能已经有了一个体验：波形在屏幕上乱跑，左右晃动，根本看不清。
这就是没有正确触发的问题。
一个类比让你秒懂触发
想象你在拍一匹奔跑的马：
没有触发：                              有触发：
每张照片的马都在画面中不同位置              每张照片的马都在画面正中央
照片1:  🐴→                              照片1:     🐴→
照片2:      🐴→                          照片2:     🐴→
照片3:          🐴→                      照片3:     🐴→
照片4:              🐴→                  照片4:     🐴→
                                          
看起来像好几匹不同的马在乱跑               看起来像同一匹马在原地踏步
→ 你无法判断马的真实形态                   → 你可以清楚看到马的每一个姿态
触发的本质就是：告诉示波器"从哪个时刻开始画波形"，让每次画图的起点完全一致，这样波形就会"叠"在一起，看起来就是稳定不动的。
> 老师笔记： 触发是新手学示波器的第一道坎。很多人买了示波器，但波形一直在跑，就以为示波器坏了或者信号有问题——90% 的情况是触发没设对。掌握了触发，你就真正"驯服"了示波器。
---
5.2 触发的工作原理——逐步拆解
5.2.1 示波器是如何采集波形的？
示波器并不是"实时"显示信号的。它的工作方式是：
┌─────────────────────────────────────────────────────┐
│  ① 等待触发条件满足                                   │
│     ↓                                                │
│  ② 触发条件满足 → 开始采集一段数据                     │
│     ↓                                                │
│  ③ 数据存入存储器（Record Length / Memory Depth）     │
│     ↓                                                │
│  ④ 将数据画到屏幕上                                   │
│     ↓                                                │
│  ⑤ 画完后 → 回到 ①，等待下一次触发                     │
│     ↓                                                │
│  ⑥ 循环往复，每秒重复几十次到几万次                    │
└─────────────────────────────────────────────────────┘
关键理解： 示波器是在*不断地"拍照-显示-拍照-显示"*，而不是连续地画线。每一次"拍照"都从一个固定的触发点开始。
5.2.2 触发的三个核心要素
触发 = 触发源 + 触发条件 + 触发模式
要素	含义	问题
触发源（Source）	用哪个通道的信号来触发？	"谁说了算？"
触发条件（Condition）	什么条件下触发？	"什么条件触发？"
触发模式（Mode）	触发条件不满足时怎么办？	"等不到怎么办？"
---
5.3 触发源（Trigger Source）
选项说明
触发源	含义
CH1	用 CH1 的信号来触发
CH2	用 CH2 的信号来触发
CH3 / CH4	用 CH3/CH4 信号触发
EXT（外部触发）	用外部触发接口的信号触发
AC Line（市电触发）	用 50Hz 市电信号触发
怎么选触发源？
基本原则：用你最关心的那个信号作为触发源。
场景 1：只看一个信号
  → 触发源 = 那个信号所在的通道
场景 2：看两个信号（如 PWM + DIR）
  → 触发源 = 你想"稳定住"的那个信号
  → 通常选 PWM（CH1），因为 PWM 是周期信号，容易触发
场景 3：看多个信号的关系
  → 触发源 = 主时钟 / 主控信号
> 工程师经验： 触发源的选择直接影响波形的稳定性。选错了触发源，波形就会"飘"。比如你用 CH2（DIR 信号，可能是恒定电平）来触发，而 DIR 一直是低电平——示波器永远等不到触发条件（因为信号不变化），波形就不刷新了。
---
5.4 边沿触发（Edge Trigger）——最基础也最常用
边沿触发是示波器的默认触发类型，90% 的测量都用它。
5.4.1 什么是"边沿"？
信号从低到高或从高到低变化的那一瞬间就是"边沿"：
上升沿（Rising Edge）：     下降沿（Falling Edge）：
  电压                       电压
   ↑                         ↑
   │     ╱                   │  ┌──────┐
   │    ╱                    │  │      │
   │   ╱ ← 上升沿            │  │      │
   │  ╱                     │  │      ╱ ← 下降沿
   │ ╱                      │  │     ╱
   │╱                       ──┘    ╱
   └──────────→ 时间        └──────────────→ 时间
5.4.2 边沿触发的完整条件
边沿触发 = 触发源 + 触发沿 + 触发电平
触发条件 = 信号在指定方向穿越触发电平的那一刻
举例：
  触发源 = CH1
  触发沿 = 上升沿
  触发电平 = 1.5V
  含义：每当 CH1 的信号从低向高穿过 1.5V 时，示波器开始采集
5.4.3 触发电平（Trigger Level）详解
触发电平是一条水平线，它在屏幕上以一条带箭头的虚线显示：
  电压
   ↑
   │  3.3V  ─────┐     ┌────
   │              │     │
   │              │     │
   │  1.5V ─ ─ ─ ┼─────┼── ← 触发电平线
   │              │     │   （屏幕右侧有一个 ▶ 标记）
   │              │     │
   │  0V  ────────┘     └────
   │                     ↑
   │                  触发点：上升沿穿过 1.5V 的位置
   └──────────────────────────→ 时间
屏幕右侧有一个三角形标记（▶ 或 ◀），显示当前触发电平的位置。
触发电平怎么调？
物理旋钮：Trigger Level 旋钮
操作	效果
顺时针旋转 Level	触发电平线上移（电压值增大）
逆时针旋转 Level	触发电平线下移（电压值减小）
触发电平设在什么位置最好？
原则：设在信号幅度的 50% 处，也就是信号跳变最陡峭的位置。
最佳位置：                    太低：                    太高：
信号中间                      可能在噪声上触发            可能触发不到
   │     ╱                      │     ╱                  │     ╱
   │    ╱                       │  ╱╲ ╱╲                │    ╱
   │   ╱ ← Level 在这里        │ ╱  ╳  ╲              │   ╱ ← 超出了
   │  ╱                        │╱    ↑   ╲             │  ╱    信号最高点
   │ ╱                         │  在噪声上触发            │╱     触发不到！
   └────────→                   └────────→                └────────→
  ✓ 稳定触发                  ✗ 可能误触发              ✗ 完全不触发
不同信号的推荐触发电平：
信号类型	幅度
3.3V 方波	0~3.3V
5V 方波	0~5V
3.3V 正弦波	±1.65V（以 1.65V 为中心）
市电 220V	±311V
> 工程师经验： 如果波形一直乱跑，90% 的原因是触发电平设在了信号范围之外。快速修复方法：旋转 Level 旋钮，让触发电平线移动到波形的中间位置。
5.4.4 触发沿的选择
上升沿触发：                 下降沿触发：
  触发点在                   触发点在
  信号变高的瞬间              信号变低的瞬间
   │     ╱                   │  ┌──────┐
   │    ╱  ↑                 │  │      │
   │   ╱   │触发点           │  │      │  ↑
   │  ╱    │                 │  │      │  │触发点
   │ ╱     │                 │  │      │  │
   └──────┘                  ──┘──────┴──┘
什么时候用上升沿 vs 下降沿？
场景	推荐
看时钟信号	上升沿
看 PWM 信号	上升沿
看下降沿触发的协议	下降沿
看负逻辑信号	下降沿
信号下降沿更干净	下降沿
> 提示： 如果上升沿触发波形不稳定，试试切换到下降沿——有时候另一边的边沿更干净。
5.4.5 触发位置（Trigger Position）
触发位置决定了触发点在屏幕上的水平位置：
触发位置在中间（默认）：
  ←─ 5格 ─→│←─ 5格 ─→
  │         │         │
  │  触发前  │  触发后  │
  │  的数据  │  的数据  │
  └─────────┴─────────┘
触发位置偏左（看更多触发后的数据）：
  │← 2格 →│←── 8格 ──→│
  │ 触发前 │   触发后    │
触发位置偏右（看更多触发前的数据）：
  │←── 8格 ──→│← 2格 →│
  │   触发前   │ 触发后 │
操作： 旋转 Horizontal Position 旋钮 可以调整触发位置。
默认设在屏幕中间最通用，你能同时看到触发点之前和之后的信息。

5.5 触发模式（Trigger Mode）——等不到触发怎么办？
5.5.1 三种模式对比
这是新手最容易困惑的部分，我详细对比：
① Auto 模式（自动触发）
┌──────────────────────────────────────────┐
│  触发条件满足？                           │
│     │                                    │
│     ├─ 是 → 正常触发，稳定显示波形         │
│     │                                    │
│     └─ 否 → 强制刷新显示（不等了！）       │
│             → 波形会在屏幕上"漂移"         │
│             → 但至少你不会看到"死屏"       │
└──────────────────────────────────────────┘
特点： 示波器会强制刷新屏幕，即使没有满足触发条件的信号。
有信号时：                 没有信号时（或触发电平设错时）：
  ┌──┐  ┌──┐  ┌──┐        ≈≈≈≈≈≈≈≈≈≈≈≈≈≈≈≈≈
  │  │  │  │  │  │        （随机噪声在漂移）
  └──┘  └──┘  └──┘
  稳定！                    不稳定，但至少有东西看
适用场景：
- 日常调试（最常用）
- 信号幅度或频率未知时
- 新手不知道怎么设置时
缺点： 当信号不满足触发条件时，Auto 模式会"假装"触发了，让你误以为信号正常。但实际上波形在漂移，并不稳定。
---
② Normal 模式（普通触发）
┌──────────────────────────────────────────┐
│  触发条件满足？                           │
│     │                                    │
│     ├─ 是 → 正常触发，稳定显示波形         │
│     │                                    │
│     └─ 否 → 保持上一次的波形不刷新         │
│             → 屏幕上显示的是"旧"波形       │
│             → 看起来像"死屏"              │
└──────────────────────────────────────────┘
特点： 只有触发条件满足才刷新，否则保持上一帧不变。
有信号且触发正确：           没有信号时：
  ┌──┐  ┌──┐  ┌──┐         ┌──┐  ┌──┐  ┌──┐
  │  │  │  │  │  │         │  │  │  │  │  │
  └──┘  └──┘  └──┘         └──┘  └──┘  └──┘
  稳定！                     不刷新（显示的是之前的波形）
                            屏幕右上角可能显示 "Trig?" 或 "Waiting..."
适用场景：
- 捕获间歇性出现的信号（如偶发的毛刺）
- 信号频率很低（如 1Hz 以下），Auto 模式等不及
- 确认触发条件是否正确（如果设错了，屏幕会"死"）
缺点： 触发条件设错时，屏幕看起来"卡住了"，新手会以为示波器坏了。
> 工程师经验： 很多人用 Normal 模式时发现屏幕"不动了"，就以为信号断了。其实可能是触发电平设错了。快速排查方法：看看屏幕上有没有 "Trig?" 或 "Auto?" 提示——如果有，说明触发条件没满足。
---
③ Single 模式（单次触发）
┌──────────────────────────────────────────┐
│  等待触发条件满足                         │
│     │                                    │
│     ├─ 满足 → 采集一次波形 → 停止！       │
│     │         → 屏幕定格                 │
│     │         → 不会自动刷新              │
│     │                                    │
│     └─ 需要手动按 Single 键才能再测一次    │
└──────────────────────────────────────────┘
特点： 只采集一次就停，像相机的单拍模式。
按 Single 键后：
  等待...等待...等待...
  → 信号来了！→ 采集一帧 → 停止
  → 屏幕定格，你可以慢慢分析
  
  要再测一次：
  → 再按 Single 键 → 重新等待
适用场景：
- 捕获一次性事件（如上电瞬间、按键按下瞬间、复位瞬间）
- 捕获偶发故障（设好触发条件后等着，一旦出现就抓住）
- 波形分析时需要"冻结"画面仔细看
操作流程：
1. 设置好触发条件
2. 按 Single 键（或 Stop/Run 切换到 Single 模式）
3. 等待事件发生
4. 事件发生后屏幕定格
5. 用光标和测量功能仔细分析
6. 分析完后，再按 Single 键重新捕获
5.5.2 三种模式速查表
模式	触发条件满足	触发条件不满足
Auto	稳定显示	强制刷新（波形漂移）
Normal	稳定显示	保持上一帧（屏幕"死"了）
Single	捕获一次就停	持续等待
> 老师笔记： 新手有一个经典困惑：
> 
> "我的信号明明在那里，为什么波形在跑？"
> → 大概率是 Auto 模式下触发电平设错了，信号穿不过触发电平线
> → 解决：旋转 Level 旋钮，让触发电平线穿过波形
> 
> "我的信号明明在那里，为什么屏幕不动了？"
> → 大概率是 Normal 模式下触发电平设错了，或者信号停了
> → 解决：先切回 Auto 模式确认信号还在，再调触发电平
> 
> 记住这个口诀：Auto 看得到在跑，Normal 看不到在等。

5.6 高级触发类型
边沿触发能解决 90% 的问题，但有些场景需要更高级的触发。
5.6.1 脉宽触发（Pulse Width Trigger）
原理： 当脉冲的宽度满足特定条件时触发。
┌──────────────────────────────────────────┐
│  触发条件：脉冲宽度 > 200ns              │
│                                           │
│  正常脉冲（100ns）：不触发                  │
│  ───┐  ┌────────────                     │
│     │  │                                 │
│     └──┘                                 │
│                                           │
│  异常脉冲（300ns）：触发！                  │
│  ───┐        ┌────────────                │
│     │        │  ↑触发点                    │
│     └────────┘                            │
└──────────────────────────────────────────┘
触发条件选项：
条件	含义
脉宽 > X	脉冲比 X 宽时触发
脉宽 < X	脉冲比 X 窄时触发
脉宽 = X	脉冲宽度等于 X 时触发
脉宽 ≠ X	脉冲宽度不等于 X 时触发
实际应用：
- 你的 PWM 应该输出 50% 占空比的方波，但偶尔出现异常窄脉冲 → 用脉宽触发捕获
- 串口通信偶尔丢了一个 bit → 用脉宽触发抓异常 bit 宽度
5.6.2 欠幅触发（Runt Trigger）
原理： 当信号穿越了一个电平但没有穿越另一个电平时触发——也就是信号"没跳到位"。
正常信号：                欠幅信号（没跳到位）：
  ┌──────┐               ┌──┐
  │      │               │  │  ← 只跳了一半
  │      │               │  │
  └──────┘               └──┘
  完整跳变               没有到达高电平阈值
  不触发                  触发！
应用：
- 总线信号偶尔没有驱动到有效电平
- 电源电压偶尔跌落
- 数字信号在阈值附近振荡
5.6.3 建立时间/保持时间触发（Setup/Hold Trigger）
原理： 当数据信号相对于时钟信号的建立时间或保持时间不满足要求时触发。
        时钟上升沿
           ↑
数据 ───╱╲──│────────────   建立时间足够 ✓ 不触发
       ↑
     数据在时钟前就稳定了
数据 ────╱╲│────────────    建立时间不够 ✗ 触发！
          ↑
        数据在时钟沿附近才变化
应用： FPGA/ASIC 时序调试——捕获时序违例。
5.6.4 超时触发（Timeout Trigger）
原理： 信号在指定时间内没有发生变化时触发。
正常：每隔 1ms 有一个脉冲
  ──┐  ┌──┐  ┌──┐  ┌──┐
    │  │  │  │  │  │
    └──┘  └──┘  └──┘
异常：某处脉冲丢失（超过 2ms 没有脉冲）
  ──┐  ┌──┐  ┌──────────────  ← 超过 2ms 没有新脉冲 → 触发！
    │  │  │  │              ↑
    └──┘  └──┘            触发点
应用： 通信协议中的超时错误、看门狗超时、中断丢失。
5.6.5 视频触发（Video Trigger）
原理： 根据视频信号的行/场同步信号触发。
应用： 调试 PAL/NTSC 视频信号、摄像头输出、VGA 信号。
5.6.6 串行总线触发
现代中高端示波器支持对通信协议的解码和触发：
协议	触发条件示例
I2C	触发特定地址、特定数据值、START/STOP 条件、ACK/NACK
SPI	触发特定 MOSI/MISO 数据值、CS 片选有效
UART/RS232	触发特定字节值、波特率错误、帧错误
CAN	触发特定 ID、特定数据
USB	触发特定包类型
实际应用举例：
你在调试 I2C 通信，想看对地址 0x50 的 EEPROM 写入 0xFF 时发生了什么：
  触发设置：
  → 类型：I2C
  → 条件：地址 = 0x50，数据 = 0xFF，方向 = 写
  
  结果：示波器会一直在等待，直到总线上出现
       对 0x50 写入 0xFF 的操作，然后自动触发并显示
       完整的 I2C 时序波形
> 工程师经验： 串行总线触发是现代示波器最有用的高级功能之一。以前调试 I2C/SPI 要手动数时钟周期、手动解码每一位数据，现在示波器自动完成。如果你有条件，强烈建议买一台支持协议解码的示波器。

5.7 触发释抑（Trigger Holdoff）
什么是触发释抑？
触发释抑是示波器在触发一次之后，拒绝再次触发的一段时间。
没有释抑：                     有释抑（300ns）：
信号每穿越一次触发电平就触发一次    触发后等待 300ns 才能再次触发
  ┌┐  ┌┐  ┌┐  ┌┐              ┌┐        ┌┐        ┌┐
  ││  ││  ││  ││              ││  忽略   ││  忽略   ││
──┘└──┘└──┘└──┘└──          ──┘└────────┘└────────┘└──
  ↑ ↑ ↑ ↑                    ↑          ↑          ↑
  每次都触发，波形可能不稳定      只在第一个上升沿触发
  
  可能触发到不同的周期上          每次都触发到同一个周期上
什么时候需要用释抑？
当信号有多个"看起来一样"的边沿，但你只想在特定的那个边沿触发时。
典型场景：
一串脉冲信号（如 SPI 的 MOSI 数据）：
  ──┐  ┌──┐  ┌─┐  ┌──┐  ┌─┐  ┌──┐
    │  │  │  │ │  │  │ │  │  │  │
    └──┘  └──┘ └──┘  └──┘ └──┘
你想在整个数据包的第一个上升沿触发，
而不是每个 bit 的上升沿都触发。
→ 设置释抑时间 = 一个完整数据包的时间长度
→ 这样触发一次后，整个包期间不会再触发
→ 下一次触发就又是下一个包的第一个上升沿
操作方法
1. 按 Trigger Menu 键
2. 找到 "释抑" 或 "Holdoff" 选项
3. 旋转旋钮调整释抑时间
4. 也可以直接输入数值

5.8 预触发/延迟触发（Pre-trigger / Delay）
预触发（Pre-trigger）
预触发让你能看到触发点之前发生了什么。
触发点
  ↓
  │←── 触发前 ──→│←── 触发后 ──→│
  │               │              │
  │  你能看到     │  你也能看到   │
  │  信号出问题之前│  信号出问题之后│
  │  的状态       │  的状态       │
  例如：你想看上电瞬间的过程
  → 触发条件设为电压超过 1V（上升沿）
  → 触发前就能看到电压从 0V 开始爬升的过程
默认情况下，示波器的存储器中触发点前后的数据都会被保留和显示。 触发位置在屏幕中间时，你能看到触发前后各 5 格的数据。
延迟触发（Delay Trigger）
延迟触发让你能跳过信号的前 N 个周期，在第 N+1 个周期才触发。
延迟 3 个周期触发：
  周期1  周期2  周期3  周期4  周期5
  ┌┐    ┌┐    ┌┐    ┌┐    ┌┐
  ││    ││    ││    ││    ││
──┘└────┘└────┘└────┘└────┘└──
  跳过       跳过       跳过  ↑
                           在这里触发！
应用： 捕获周期性信号中第 N 个周期的异常。比如你的 PWM 信号每 100 个周期出现一次毛刺，设延迟 99 个周期，就能在第 100 个周期触发。

5.9 触发系统故障排查速查表
现象	可能原因
波形一直左右漂移	触发电平不在信号范围内
波形有时稳定有时漂移	信号上有噪声，触发电平在噪声附近
屏幕完全不动（Normal 模式）	触发条件不满足或信号消失
波形跳动不稳定	触发沿选错了
波形时有时无	信号间歇性出现
看不到信号变化前的状态	触发位置偏了
每次触发的位置不一样	信号有多个相似的边沿
屏幕显示 "Trig?"	触发条件未满足

5.10 模块 5 小结
知识点	核心内容
触发的作用	固定波形起点，让波形稳定显示
三要素	触发源 + 触发条件 + 触发模式
触发源	选你最关心的信号通道
边沿触发	最常用；上升沿/下降沿穿越触发电平时触发
触发电平	设在信号幅度的 50% 处最稳定
Auto 模式	等不到也刷新，适合日常调试
Normal 模式	等不到就不刷新，适合捕获间歇信号
Single 模式	触发一次就停，适合捕获一次性事件
脉宽触发	捕获异常宽窄脉冲
串行总线触发	按协议内容触发（I2C/SPI/UART 等）
触发释抑	触发后拒绝再次触发的一段时间
> 老师寄语： 触发是示波器最值得深入理解的功能。边沿触发只是起点——当你学会使用脉宽触发、释抑、单次触发等高级功能后，你能捕获到那些"转瞬即逝"的异常信号，这是排查间歇性 bug 的最强武器。建议你用信号发生器产生各种信号，反复练习触发设置，直到形成直觉。
