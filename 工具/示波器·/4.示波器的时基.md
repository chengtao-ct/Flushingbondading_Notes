4.1 为什么时间参数和电压参数同等重要？
同学，模块 3 我们讲完了电压参数——解决了信号跳多高的问题。现在模块 4 我们要解决信号跳多快的问题。
在数字电路中，时间参数往往比电压参数更致命。
举个例子：
> 一个 100MHz 的时钟信号，电压幅度完美（3.3V），但如果上升时间太慢（比如从 10ns 变成了 30ns），芯片可能就无法正确采样数据，整个系统直接崩溃。
电压决定"能不能工作"，时间决定"能不能稳定工作"。

4.2 Timebase（时基）——示波器水平轴的核心
什么是时基？
时基（Timebase），也叫 Time/Div 或 Horizontal Scale，是示波器屏幕上每一格（division）代表的时间长度。
         Time/Div = 1ms/div
              │
              ↓
  电压        1格=1ms
   ↑    ┌────┬────┬────┬────┬────┬────┬────┬────┬────┬────┐
   │    │    │    │    │    │    │    │    │    │    │    │
   │    │    │    │    │    │    │    │    │    │    │    │
   │    │    │    │    │    │    │    │    │    │    │    │
   └────┴────┴────┴────┴────┴────┴────┴────┴────┴────┴──→ 时间
        0   1ms  2ms  3ms  4ms  5ms  6ms  7ms  8ms  9ms  10ms
旋钮操作：
操作	效果	类比
顺时针旋转 Time/Div	每格时间变短 → 波形被拉伸 → 看到更多细节	照相机放大拍特写
逆时针旋转 Time/Div	每格时间变长 → 波形被压缩 → 看到更多周期	照相机缩小拍全景
时基的常见范围
范围	每格代表的时间	典型应用场景
极慢	1s/div ~ 50s/div	电源上电过程、电池充放电
慢	1ms/div ~ 500ms/div	电机启动、传感器缓慢变化
中等	1μs/div ~ 1ms/div	音频信号（20Hz~20kHz）、串口通信（UART）
快	1ns/div ~ 1μs/div	SPI/I2C 通信、PWM 信号
极快	100ps/div ~ 1ns/div	高速信号（USB、PCIe、DDR）
屏幕总时间的计算
示波器屏幕水平方向通常有 10 格，所以：
$$\text{屏幕总时间} = \text{Time/Div} \times 10$$
Time/Div 设置	屏幕总时间	能看到什么
1s/div	10 秒	10 秒内的信号变化
1ms/div	10 毫秒	约 1 个 100Hz 的完整周期
1μs/div	10 微秒	约 10 个 1MHz 的完整周期
1ns/div	10 纳秒	约 10 个 100MHz 的完整周期
> 老师笔记： 新手最常见的操作错误——Time/Div 设置不对：
> - 设得太快（每格时间太短）→ 屏幕上只看到信号的一小段，看不到完整波形
> - 设得太慢（每格时间太长）→ 波形被压缩成一条线，看不清任何细节
>
> 正确做法： 先把 Time/Div 调到能看到 2~4 个完整波形的档位，再根据需要放大或缩小。

4.3 Period（周期）和 Frequency（频率）——回顾与深入
在模块 2 我们已经讲过 f = 1/T，这里从示波器操作的角度深入。
在示波器上如何测量周期？
电压
  ↑
  │    ╱╲          ╱╲
  │   ╱  ╲        ╱  ╲
  │  ╱    ╲      ╱    ╲
  │ ╱      ╲    ╱      ╲
  │╱        ╲──╱        ╲
  └──────────────────────────→ 时间
  ↑                   ↑
  起点               终点（下一个相同位置）
  │←──── 周期 T ────→│
三种方法：
方法	操作	精度	适用场景
数格子	数一个完整波形占几格 × Time/Div	低（±1格误差）	快速估算
自动测量	按 Measure → 选 Period / Frequency	中	日常调试
光标测量	按 Cursor → 选 Time 光标 → 放在两个相邻同相位点	高	精确测量
数格子的练习：
Time/Div = 2ms/div
  │    │    │    │    │    │    │    │    │    │    │
  │    │    │    │    │    │    │    │    │    │    │
  │    ╱╲       ╱╲       ╱╲       ╱╲       ╱╲       │
  │   ╱  ╲     ╱  ╲     ╱  ╲     ╱  ╲     ╱  ╲    │
  │──╱    ╲───╱    ╲───╱    ╲───╱    ╲───╱    ╲───│
  │        ╲ ╱        ╲ ╱        ╲ ╱        ╲     │
  │         ╲          ╲          ╲          ╲     │
  │    │    │    │    │    │    │    │    │    │    │
      0   2ms  4ms  6ms  8ms  10ms 12ms 14ms 16ms 18ms 20ms
- 一个完整波形占 4 格
- T = 4 × 2ms = 8ms
- f = 1/8ms = 125 Hz
频率的工程直觉——必须形成的速算能力
看到的现象	推算过程	结果
屏幕上 1 格 = 1 个完整波，Time/div = 1ms	T = 1ms	f = 1 kHz
屏幕上 1 格 = 1 个完整波，Time/div = 1μs	T = 1μs	f = 1 MHz
屏幕上 5 格 = 1 个完整波，Time/div = 2μs	T = 10μs	f = 100 kHz
屏幕上 2 格 = 1 个完整波，Time/div = 5ns	T = 10ns	f = 100 MHz
> 工程师经验： 在调试中你不需要每次都按计算器。多练习，形成直觉：
> - 1μs 周期 → 1MHz（最基本，必须秒出）
> - 10ns 周期 → 100MHz
> - 100ms 周期 → 10Hz

4.4 Rise Time（上升时间）和 Fall Time（下降时间）
这是数字电路中最重要的两个时间参数。
定义
电压
  ↑  100%  ─────────────────────
  │         ╱╲ ← 过冲
  │        ╱  ╲
  │       ╱    ╲────────────────── ← 90% (高电平阈值)
  │      ╱     │
  │     ╱      │
  │    ╱       │ ← 上升时间
  │   ╱        │
  │  ╱         │
  │ ╱          │
  │╱           │
  │────────────│← 10% (低电平阈值)
  └──────────────────────────────────→ 时间
       ← tr →
Rise Time（tr，上升时间）： 信号从 10% 到 90% 的幅度所经历的时间。
Fall Time（tf，下降时间）： 信号从 90% 到 10% 的幅度所经历的时间。
电压
  ↑  100% ────────────────────────
  │       │
  │  90% ─┤                      │
  │       │╲                     │
  │       │ ╲                    │
  │       │  ╲  ← 下降时间       │
  │       │   ╲                  │
  │  10% ─┤    ╲                 │
  │       │     ╲                │
  │    0% ───────╲───────────────│
  └──────────────────────────────────→ 时间
            ←─ tf ─→
为什么是 10%~90% 而不是 0%~100%？
因为实际信号在 0% 附近和 100% 附近往往有噪声、振铃、过冲，很难精确确定真正的起点和终点。10%~90% 的定义避开了这些区域，是工业标准。
上升时间与带宽的关系（重要公式）
$$BW \approx \frac{0.35}{t_r}$$
其中：
- BW = 示波器带宽（Hz）
- tr = 信号的上升时间（s）
反过来：
$$t_r \approx \frac{0.35}{BW}$$
实际意义：
示波器带宽	能准确测量的最小上升时间	适用场景
100 MHz	~3.5 ns	Arduino、基础数字电路
350 MHz	~1.0 ns	STM32、SPI/I2C 中速信号
1 GHz	~350 ps	USB 2.0、LVDS
2 GHz	~175 ps	DDR3、PCIe、高速串行
4 GHz+	~87 ps	DDR4/5、USB 3.x、SerDes
> 老师笔记： 这是选择示波器带宽的核心依据。一个常见的工程经验法则：示波器带宽至少是信号最高频率分量的 5 倍，或者示波器的上升时间至少是信号上升时间的 1/3 到 1/5。
>
> 举例：你的信号上升时间是 5ns，你需要示波器上升时间 ≤ 5/3 ≈ 1.67ns，对应带宽 ≥ 0.35/1.67ns ≈ 210 MHz。所以你需要至少一台 200MHz 的示波器。

4.5 Pulse Width（脉宽）和 Duty Cycle（占空比）
Pulse Width（tw，脉宽）
定义： 脉冲信号处于有效电平的时间长度。
对于正逻辑（高电平有效）：
电压
  ↑   ┌───────────┐            ┌───────────┐
  │   │           │            │           │
  │   │    ← tw → │            │    ← tw → │
  │   │           │            │           │
───┘   └───────────┘────────────┘           └─────
对于负逻辑（低电平有效）：
电压
  ↑                       ┌───────────────────────
  │                       │
───┐           ┌──────────┘           ┌──────────
  │           │                        │
  │   ← tw →  │           ← tw →       │
Duty Cycle（占空比）
定义： 脉宽占整个周期的百分比。
$$\text{Duty Cycle} = \frac{t_w}{T} \times 100\%$$
常见的占空比场景：
占空比	应用	说明
50%	时钟信号（Clock）	高低电平时间相等，标准方波
1%~99%	PWM 调光/调速	通过改变占空比控制平均电压
< 1%	超声波脉冲、雷达	极窄脉冲，用于测距/探测
固定比例	通信协议	如 I2C 的 SCL、UART 的起始位
PWM 调光的直观理解：
占空比 10%：
──┐  ┌──────────────────────────────────┐  ┌──────
  │  │                                  │  │
  └──┘                                  └──┘
  LED 很暗（10% 时间亮）
占空比 50%：
──┐        ┌────────────────────────┐        ┌──
  │        │                        │        │
  └────────┘                        └────────┘
  LED 中等亮度
占空比 90%：
────────────────┐  ┌────────────────────────────┐  ┌
                │  │                            │  │
                └──┘                            └──
  LED 很亮（90% 时间亮）
> 工程师经验： PWM 频率和占空比是两个独立参数：
> - 频率决定 PWM 的"速度"（人眼看不到闪烁需要 > 100Hz）
> - 占空比决定"有效电压"（平均电压 = VDD × 占空比）
>
> 例如：5V 系统，PWM 频率 1kHz，占空比 40%
> - LED 两端平均电压 = 5V × 40% = 2V
> - 用万用表测会读到约 2V
> - 用示波器能看到 1kHz 的方波，高电平 5V，低电平 0V

4.6 Setup Time（建立时间）和 Hold Time（保持时间）
这两个参数主要用于数字电路时序分析，是芯片能否正常工作的关键。
定义
          时钟边沿(上升沿)
              ↑
              │
数据信号： ────┤───────┤───────────────
              │       │
              │← 建立 →│← 保持 →│
              │  时间   │  时间   │
              │(tsu)   │(th)    │
              │       │
时钟信号： ────╱───────│───────────────
            ╱
           ╱
Setup Time（tsu，建立时间）： 数据必须在时钟边沿到来之前提前稳定的最短时间。
Hold Time（th，保持时间）： 数据在时钟边沿到来之后必须保持稳定的最短时间。
违反时序的后果
正确时序：              建立时间违反：
数据 ─────┤──────┤──   数据 ──────╱╲──┤──────
         │      │                │   │
时钟 ────╱──────│──            ──╱──────│──
        ↑(正确采样)              ↑(可能采样到错误值)
违反类型	后果
建立时间不足	芯片采样到旧数据或中间态
保持时间不足	芯片采样到新数据或中间态
两者都违反	100% 数据错误
> 工程师经验： 时序违例是 FPGA/ASIC 设计中最常见的 bug。用示波器同时抓取数据信号和时钟信号（双通道），就能直接观察到建立时间和保持时间是否满足要求。这就是为什么至少需要双通道示波器的原因。

4.7 Propagation Delay（传播延迟）
定义： 信号从电路的一个点传输到另一个点所需要的时间。
输入信号：  ───┐     ┌──────────────────
            │     │
            └─────┘
            ↑
            │←── 传播延迟 tpd ──→│
            ↑                    ↑
输出信号：  ─────────────────────┐     ┌──
                                │     │
                                └─────┘
常见来源：
延迟来源	典型延迟量	说明
逻辑门延迟	1~10 ns	一个反相器/与非门的固有延迟
PCB 走线延迟	~6.7 ns/m（FR4板材）	信号在铜线上传播的速度
光耦延迟	50 ns ~ 数 μs	光耦隔离器
电缆延迟	~5 ns/m	同轴电缆/普通导线
ADC 转换延迟	数 μs ~ 数 ms	模数转换时间
> 老师笔记： 传播延迟在高频电路中尤其重要。一个 1GHz 的信号，周期只有 1ns——如果走线延迟就有 1ns，那信号还没到达目的地，下一个周期就已经开始了。这就是为什么高速 PCB 设计需要做时序仿真（Timing Simulation）。

4.8 Glitch（毛刺）和 Bounce（抖动/弹跳）
Glitch（毛刺）
定义： 信号上出现的短暂、非预期的电压尖峰。
正常信号：    ───┐          ┌──────────
              │          │
              └──────────┘
有毛刺：      ───┐  ╱╲      ┌──────────
              │  │  │      │
              └──╱  ╲──────┘
                  ↑
               毛刺（几ns宽）
常见原因：
- 电磁干扰（EMI）
- 信号串扰（Crosstalk）
- 电源噪声耦合
- 逻辑竞争（Race Condition）
Bounce / Ringing（振铃/弹跳）
定义： 信号边沿之后出现的阻尼振荡。
理想上升沿：    ╱│
              ╱ │
             ╱  │
实际上升沿：   ╱╲│
             ╱  ╲╱╲
            ╱      ╲
           ╱        ╲──
                     ↑
                   振铃
常见原因：
- 阻抗不匹配导致的信号反射
- PCB 走线的寄生电感和电容
- 过长的引线/飞线
> 工程师经验： 消除振铃的方法：
> 1. 串联电阻（通常 22Ω~47Ω），靠近信号源端放置
> 2. 匹配终端电阻（并联）
> 3. 减小走线长度
> 4. 使用合适的 PCB 层叠和阻抗控制
